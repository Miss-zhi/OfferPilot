### 1.1 Logback 配置文件（logback-spring.xml）

> **说明**：Spring Boot 默认使用 Logback，`logback-spring.xml` 放在 `src/main/resources` 下。
> 生产环境输出**结构化 JSON 日志**（便于 ELK / Grafana Loki 采集），开发环境输出**彩色控制台日志**（便于人工调试）。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>

    <!-- ============================================================ -->
    <!-- 从 application.yml 读取属性                                   -->
    <!-- ============================================================ -->
    <springProperty scope="context" name="LOG_PATH"       source="logging.file.path"       defaultValue="./logs"/>
    <springProperty scope="context" name="LOG_FILE"       source="logging.file.name"       defaultValue="offerpilot"/>
    <springProperty scope="context" name="APP_NAME"       source="spring.application.name" defaultValue="offerpilot"/>
    <springProperty scope="context" name="ACTIVE_PROFILE" source="spring.profiles.active"  defaultValue="dev"/>

    <!-- ============================================================ -->
    <!-- 控制台 Appender（开发环境：彩色 + 简洁格式）                    -->
    <!-- ============================================================ -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{HH:mm:ss.SSS} %highlight(%-5level) [%thread] %cyan(%logger{36}) - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- ============================================================ -->
    <!-- 文件 Appender（JSON 结构化日志，便于日志平台采集）               -->
    <!-- ============================================================ -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/${LOG_FILE}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/${LOG_FILE}.%d{yyyy-MM-dd}.%i.log.gz</fileNamePattern>
            <timeBasedFileNamingAndTriggeringPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedFNATP">
                <maxFileSize>10MB</maxFileSize>
            </timeBasedFileNamingAndTriggeringPolicy>
            <maxHistory>7</maxHistory>
            <totalSizeCap>500MB</totalSizeCap>
        </rollingPolicy>
        <encoder class="net.logstash.logback.encoder.LogstashEncoder">
            <customFields>{"app":"${APP_NAME}","env":"${ACTIVE_PROFILE}"}</customFields>
            <includeMdcKeyName>userId</includeMdcKeyName>
            <includeMdcKeyName>sessionId</includeMdcKeyName>
            <includeMdcKeyName>traceId</includeMdcKeyName>
        </encoder>
    </appender>

    <!-- ============================================================ -->
    <!-- 异步 Appender（避免磁盘 I/O 阻塞业务线程）                     -->
    <!-- ============================================================ -->
    <appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
        <appender-ref ref="FILE"/>
        <queueSize>512</queueSize>
        <discardingThreshold>0</discardingThreshold>
        <neverBlock>true</neverBlock>
    </appender>

    <!-- ============================================================ -->
    <!-- 按 Profile 切换策略                                           -->
    <!-- ============================================================ -->

    <!-- 开发/测试环境：控制台彩色输出 + JSON 文件（本地调试） -->
    <springProfile name="dev,test">
        <root level="INFO">
            <appender-ref ref="CONSOLE"/>
            <appender-ref ref="ASYNC_FILE"/>
        </root>
        <logger name="com.tutorial.offerpilot" level="DEBUG"/>
        <logger name="org.springframework.security" level="DEBUG"/>
        <logger name="org.springframework.web" level="INFO"/>
        <logger name="org.hibernate.SQL" level="WARN"/>
    </springProfile>

    <!-- 生产环境：仅 JSON 文件（日志平台通过 Filebeat 采集） -->
    <springProfile name="prod">
        <root level="INFO">
            <appender-ref ref="ASYNC_FILE"/>
        </root>
        <logger name="com.tutorial.offerpilot" level="INFO"/>
        <logger name="org.springframework.security" level="WARN"/>
    </springProfile>

</configuration>
```

**关键设计决策：**

| 决策 | 说明 |
|------|------|
| JSON 结构化日志 | 使用 `LogstashEncoder` 输出 JSON → Filebeat 采集 → ELK/Loki 可视化查询 |
| 异步 Appender | `AsyncAppender` 包装文件 Appender，日志写入不阻塞业务线程 |
| Profile 区分 | `dev/test` 控制台彩色 + 文件，`prod` 仅文件（减少控制台 I/O） |
| MDC 上下文 | `userId`、`sessionId`、`traceId` 自动注入每条日志，实现请求级链路追踪 |
| 按天滚动 + 压缩 | `%d{yyyy-MM-dd}` 按天分文件，`.gz` 自动压缩，`maxHistory=7` 保留 7 天 |
| 总容量上限 | `totalSizeCap=500MB`，超出后自动删除最旧文件 |
```

## 3. Service 层关键代码

### 3.1 AgentFactory

```java
@Component
public class AgentFactory {

    private final AgentScopeProperties properties;
    private final Model model;
    private final StringRedisTemplate redisTemplate;
    private final UserMemoryService userMemoryService;   // ← 注入记忆服务

    // 13 个 @Tool 组件（Spring 单例，必须线程安全 — 详见《02-系统架构设计说明书》§工具定义 - Tool 线程安全规范）
    private final ResumeParseTool resumeParseTool;
    private final ResumeEvaluateTool resumeEvaluateTool;
    private final QuestionSearchTool questionSearchTool;
    private final AnswerSearchTool answerSearchTool;
    private final CompanySearchTool companySearchTool;
    private final AnswerAnalyzeTool answerAnalyzeTool;
    private final AudioTranscribeTool audioTranscribeTool;
    private final MockInterviewTool mockInterviewTool;
    private final ResourceSearchTool resourceSearchTool;
    private final ProgressTrackTool progressTrackTool;
    private final SalaryTool salaryTool;

    /** Caffeine 有界缓存：最多 500 个 Agent，30 分钟未访问自动淘汰 */
    private final Cache<String, HarnessAgent> agentPool = Caffeine.newBuilder()
        .maximumSize(500)
        .expireAfterAccess(Duration.ofMinutes(30))
        .removalListener((key, agent, cause) -> {
            log.info("Evicted agent: userId={}, cause={}", key, cause);
            // TODO: 淘汰时可执行清理逻辑（关闭 MCP 连接、flush 未持久化状态等）
        })
        .build();

    public AgentFactory(AgentScopeProperties properties,
                        StringRedisTemplate redisTemplate,
                        UserMemoryService userMemoryService,
                        ResumeParseTool resumeParseTool,
                        ResumeEvaluateTool resumeEvaluateTool,
                        QuestionSearchTool questionSearchTool,
                        AnswerSearchTool answerSearchTool,
                        CompanySearchTool companySearchTool,
                        AnswerAnalyzeTool answerAnalyzeTool,
                        AudioTranscribeTool audioTranscribeTool,
                        MockInterviewTool mockInterviewTool,
                        ResourceSearchTool resourceSearchTool,
                        ProgressTrackTool progressTrackTool,
                        SalaryTool salaryTool) {
        this.properties = properties;
        this.redisTemplate = redisTemplate;
        this.userMemoryService = userMemoryService;
        this.model = buildModel();
        this.resumeParseTool = resumeParseTool;
        this.resumeEvaluateTool = resumeEvaluateTool;
        this.questionSearchTool = questionSearchTool;
        this.answerSearchTool = answerSearchTool;
        this.companySearchTool = companySearchTool;
        this.answerAnalyzeTool = answerAnalyzeTool;
        this.audioTranscribeTool = audioTranscribeTool;
        this.mockInterviewTool = mockInterviewTool;
        this.resourceSearchTool = resourceSearchTool;
        this.progressTrackTool = progressTrackTool;
        this.salaryTool = salaryTool;
    }

    public HarnessAgent getOrCreateAgent(String userId) {
        return agentPool.get(userId, this::buildAgent);
    }

    private HarnessAgent buildAgent(String userId) {
        Path userWorkspace = Path.of(properties.getAgent().getWorkspace(), "users", userId);
        Files.createDirectories(userWorkspace);

        // ============================================================
        // 主 Agent Toolkit：13 个业务工具全部注册（子Agent按白名单筛选继承）
        // ============================================================
        // 架构原因：AgentScope v2 的 SubagentsMiddleware 在 spawn 子Agent时，
        // 从主Agent的Toolkit中按SubagentDeclaration.tools()白名单筛选工具，
        // 为每个子Agent构建独立的过滤Toolkit副本。因此主Agent必须持有全部工具。
        // 但主Agent的SysPrompt严禁它直接调用业务工具（详见《02-系统架构设计说明书》）。
        Toolkit toolkit = new Toolkit();
        toolkit.registerTool(resumeParseTool);
        toolkit.registerTool(resumeEvaluateTool);
        toolkit.registerTool(questionSearchTool);
        toolkit.registerTool(answerSearchTool);
        toolkit.registerTool(companySearchTool);
        toolkit.registerTool(answerAnalyzeTool);
        toolkit.registerTool(audioTranscribeTool);
        toolkit.registerTool(mockInterviewTool);
        toolkit.registerTool(resourceSearchTool);
        toolkit.registerTool(progressTrackTool);
        toolkit.registerTool(salaryTool);

        // 元工具：reset_equipped_tools（Agent可动态启用/禁用工具）
        toolkit.registerMetaTool();

        // 子 Agent 定义见《02-系统架构设计说明书》中'多 Agent 协作设计'章节
        // ...

        return HarnessAgent.builder()
            .name("offerpilot_" + userId)
            .sysPrompt("你是 OfferPilot 面试诊断助手的调度中心...\n" +
                "\n" +
                "IMPORTANT — Tool output interpretation:\n" +
                "- When you call `generate_next_question`, it returns guidance (not a ready question).\n" +
                "  You MUST read the guidance and generate an appropriate interview question yourself.\n" +
                "- When you call `analyze_answer`, it returns the raw Q&A plus evaluation guidance.\n" +
                "  You MUST generate the actual scores, highlights, weaknesses, and suggestions yourself.\n" +
                "- When you call `evaluate_resume`, it returns the resume text plus evaluation guidance.\n" +
                "  You MUST generate the actual score, strengths, weaknesses, and suggestions yourself.\n" +
                "- For all guidance-based tools, never echo the raw guidance to the user.\n" +
                "  Always transform it into natural, polished output.")
            .model(model)
            .workspace(userWorkspace)
            .toolkit(toolkit)
            .subagent(resumeAgent)
            .subagent(techEvalAgent)
            .subagent(exprEvalAgent)
            .subagent(mockAgent)
            .subagent(companyAgent)
            .subagent(studyAgent)
            .subagent(salaryAgent)
            .stateStore(new RedisAgentStateStore(redisTemplate, sessionTtl))
            .permissionContext(buildPermissions())
            .compaction(CompactionConfig.builder().maxTokens(8000).build())
            .memory(MemoryConfig.builder().build())
            .middleware(new TokenMonitorMiddleware())                       // Token 用量监控
            .middleware(new CostControlMiddleware(costTracker))             // 成本追踪 + 预算熔断
            .middleware(new MemoryInjectMiddleware(userMemoryService))      // ← 每次推理前动态注入用户记忆
            .enablePlanMode()       // 计划模式：支持断点续聊 + todo 管理
            .enableTaskList()        // 注册 todo_write 元工具
            .build();
    }
}
```

注意：用 Caffeine 替代 ConcurrentHashMap。ConcurrentHashMap 无限增长会导致 OOM，Caffeine 提供有界缓存 + 自动淘汰策略，是企业项目的标准做法。Maven 需引入 `com.github.ben-manes.caffeine:caffeine`。

### 3.3 AuthService（认证服务）

```java
@Service
public class AuthService {

    private final AppUserRepository userRepository;
    private final PasswordEncoder passwordEncoder;
    private final JwtTokenProvider jwtTokenProvider;

    public AuthResponse register(@Valid RegisterRequest req) {
        if (userRepository.findByUsername(req.getUsername()).isPresent()) {
            throw new DuplicateUserException();                 // 409
        }
        AppUser user = new AppUser();
        user.setUserId(generateUserId());
        user.setUsername(req.getUsername());
        user.setPasswordHash(passwordEncoder.encode(req.getPassword()));
        user.setEmail(req.getEmail());
        user.setRole(UserRole.USER);
        user.setEnabled(true);
        userRepository.save(user);

        String token = jwtTokenProvider.generateToken(user.getUsername());
        return new AuthResponse(token, user.getUserId(), user.getUsername(), UserRole.USER);
    }

    public AuthResponse login(@Valid LoginRequest req) {
        AppUser user = userRepository.findByUsername(req.getUsername())
            .orElseThrow(InvalidCredentialsException::new);     // 401
        if (!passwordEncoder.matches(req.getPassword(), user.getPasswordHash())) {
            throw new InvalidCredentialsException();            // 401
        }
        if (!user.getEnabled()) {

        user.setLastLoginAt(Instant.now());
        userRepository.save(user);

        String token = jwtTokenProvider.generateToken(user.getUsername());
        return new AuthResponse(token, user.getUserId(), user.getUsername(), user.getRole());
    }

### 3.5 ChatController

```java
@RestController
@RequestMapping("/api/v1/offerpilot")
public class ChatController {

    private final AgentFactory agentFactory;
    private final RateLimitService rateLimitService;

    /** SSE 流式对话（Spring MVC 用 SseEmitter） */
    @PostMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter chatStream(
            @RequestBody @Valid ChatRequest request,
            @AuthenticationPrincipal UserDetails currentUser) {

        String userId = getUserIdFromUserDetails(currentUser);

        // 限流检查
        if (!rateLimitService.tryAcquireDialogue(userId)) {
            throw new RateLimitException("对话频率过高，请稍后再试");
        }

        SseEmitter emitter = new SseEmitter(120_000L);  // 2 分钟超时
        HarnessAgent agent = agentFactory.getOrCreateAgent(userId);

        RuntimeContext ctx = RuntimeContext.builder()
            .userId(userId)
            .sessionId(request.getSessionId() == null ? generateSessionId(userId) : request.getSessionId())
            .build();

        // 异步执行 Agent，将事件推送到 SseEmitter
        agent.streamEvents(request.getMessage(), ctx)
            .subscribe(
                event -> {
                    try {
                        emitter.send(SseEmitter.event()
                            .name(event.getType())
                            .data(event.getData()));
                    } catch (IOException e) {
                        emitter.completeWithError(e);
                    }
                },
                emitter::completeWithError,
                emitter::complete
            );

        return emitter;
    }

    /** 同步对话 */
    @PostMapping("/chat")
    public ResponseEntity<ApiResponse<ChatResponse>> chat(
            @RequestBody @Valid ChatRequest request,
            @AuthenticationPrincipal UserDetails currentUser) {

        String userId = getUserIdFromUserDetails(currentUser);
        HarnessAgent agent = agentFactory.getOrCreateAgent(userId);

        RuntimeContext ctx = RuntimeContext.builder()
            .userId(userId)
            .sessionId(request.getSessionId())
            .build();

        Msg result = agent.call(request.getMessage(), ctx).block();
        ChatResponse response = new ChatResponse(result.getTextContent(), ctx.getSessionId());
        return ResponseEntity.ok(ApiResponse.success(response));
    }
}
```

注意：
- SSE 使用 `agent.streamEvents().subscribe()` 推送事件到 SseEmitter（Flux 是冷发布者，必须 subscribe 才能触发）
- 同步接口使用 `agent.call().block()` 阻塞等待 `Mono<Msg>` 完成，用 `ApiResponse<T>` 统一包装
- 新增 `@Valid` 参数校验
- 新增限流检查
- **用户记忆不再由 Controller 层拼接**：记忆通过 `MemoryInjectMiddleware` 在每次推理前从 RuntimeContext 中提取 userId 动态加载（详见本文档 §4.3 MemoryInjectMiddleware）
- `RuntimeContext.userId` 贯穿整个调用链：Controller 设置 → Agent 推理 → Middleware → Tool 执行，多租户隔离的关键

@Service
public class RateLimitService {

    private final StringRedisTemplate redis;

    /** 对话限流：每分钟最多 30 次 */
    public boolean tryAcquireDialogue(String userId) {
        String key = "ratelimit:chat:" + userId;
        Long count = redis.opsForValue().increment(key);
        if (count == 1) {
            redis.expire(key, 60, TimeUnit.SECONDS);
        }
        return count != null && count <= 30;
    }

    /** 上传限流：每分钟最多 10 次 */
    public boolean tryAcquireUpload(String userId) {
        String key = "ratelimit:upload:" + userId;
        Long count = redis.opsForValue().increment(key);
        if (count == 1) {
            redis.expire(key, 60, TimeUnit.SECONDS);
        }
        return count != null && count <= 10;
    }
}

## 4. Middleware 示例（v2 洋葱模式）

AgentScope v2 的 `MiddlewareBase` 是接口，提供 5 个插桩点，采用洋葱模式（必须调用 `next.apply(input)` 才能继续链）：

| 钩子 | 阶段 | 返回值 | 典型用途 |
|:---|:---|:---|:---|
| `onAgent` | 整轮 Agent 调用 | `Flux<AgentEvent>` | 全链路追踪 |
| `onSystemPrompt` | 系统提示词变换 | `Mono<String>` | 动态注入上下文 |
| `onReasoning` | LLM 推理阶段 | `Flux<AgentEvent>` | Token 统计 / 耗时监控 |
| `onActing` | 工具执行阶段 | `Flux<AgentEvent>` | 工具调用审计 |
| `onModelCall` | 原始模型 API 调用 | `Flux<AgentEvent>` | 按调用计费 / 模型熔断 |

### 4.1 TokenMonitorMiddleware — Token 用量监控

```java
import io.agentscope.core.agent.Agent;
import io.agentscope.core.agent.RuntimeContext;
import io.agentscope.core.event.AgentEvent;
import io.agentscope.core.middleware.MiddlewareBase;
import io.agentscope.core.middleware.ReasoningInput;
import lombok.extern.slf4j.Slf4j;
import reactor.core.publisher.Flux;

import java.util.function.Function;

/**
 * Token 消耗监控中间件。
 * 记录每次 Agent 推理的 token 用量和响应耗时，用于成本控制和服务优化。
 */
@Slf4j
public class TokenMonitorMiddleware implements MiddlewareBase {

    @Override
    public Flux<AgentEvent> onReasoning(
            Agent agent,
            RuntimeContext ctx,
            ReasoningInput input,
            Function<ReasoningInput, Flux<AgentEvent>> next) {

        long startTime = System.currentTimeMillis();

        return next.apply(input)
            .doFinally(signal -> {
                long elapsed = System.currentTimeMillis() - startTime;
                // input.getMessages() 可估算输入 token，此处使用 UsageTracker 获取精确数据
                log.info("[TokenMonitor] userId={}, sessionId={}, elapsedMs={}",
                         ctx.getUserId(), ctx.getSessionId(), elapsed);
            });
    }
}
```

### 4.2 CostControlMiddleware — 模型调用成本控制

`onModelCall` 是比 `onReasoning` 更底层的钩子——直接拦截原始模型 API 调用，可获取精确的输入/输出 token 数，实现按调用计费和预算熔断。

```java
import io.agentscope.core.agent.Agent;
import io.agentscope.core.agent.RuntimeContext;
import io.agentscope.core.event.AgentEvent;
import io.agentscope.core.middleware.MiddlewareBase;
import io.agentscope.core.middleware.ModelCallInput;
import lombok.extern.slf4j.Slf4j;
import reactor.core.publisher.Flux;

import java.util.function.Function;

/**
 * 模型调用成本控制中间件。
 * 在每次原始模型 API 调用时追踪 token 消耗，超出预算时熔断。
 */
@Slf4j
public class CostControlMiddleware implements MiddlewareBase {

    private final CostTracker costTracker;

    public CostControlMiddleware(CostTracker costTracker) {
        this.costTracker = costTracker;
    }

    @Override
    public Flux<AgentEvent> onModelCall(
            Agent agent,
            RuntimeContext ctx,
            ModelCallInput input,
            Function<ModelCallInput, Flux<AgentEvent>> next) {

        long startTime = System.currentTimeMillis();

        return next.apply(input)
            .doOnComplete(() -> {
                long latency = System.currentTimeMillis() - startTime;
                String userId = ctx.getUserId();

                // 检查预算
                if (costTracker.isBudgetExceeded(userId)) {
                    log.warn("[CostControl] Budget exceeded! userId={}", userId);
                    // 实际熔断需配合 BudgetExceededException + GlobalExceptionHandler
                }

                log.info("[CostControl] userId={}, latencyMs={}", userId, latency);
            });
    }
}
```

### 4.3 MemoryInjectMiddleware — 用户记忆动态注入（替代静态拼接）

**为什么不用 ChatController 中静态拼接 system prompt：** 存在两个致命缺陷——(1) 记忆在 `AgentFactory.buildAgent()` 时一次性注入，Agent 被 Caffeine 缓存 30 分钟期间，用户记忆变更（如 Agent 识别出新薄弱点写入 DB）无法被 Agent 感知；(2) 当前实现中 `buildAgent(userId)` 实际没有使用 `ChatController` 中的 `userMemory` 变量，记忆功能实际未生效。

**正确方案**：使用 AgentScope v2 的 `onSystemPrompt` middleware 钩子，在**每次推理前**动态从 DB 加载最新记忆并拼接到 system prompt 末尾。

```java
import io.agentscope.core.agent.Agent;
import io.agentscope.core.agent.RuntimeContext;
import io.agentscope.core.middleware.MiddlewareBase;
import lombok.extern.slf4j.Slf4j;
import reactor.core.publisher.Mono;

import java.util.function.Function;

/**
 * 用户记忆动态注入中间件。
 * 
 * 在每次 Agent 推理前从 DB 加载当前用户的最新记忆，
 * 拼接到 system prompt 末尾。解决了静态注入的时效性问题和
 * Agent 缓存期间记忆不更新的竞态。
 */
@Slf4j
public class MemoryInjectMiddleware implements MiddlewareBase {

    private final UserMemoryService userMemoryService;

    public MemoryInjectMiddleware(UserMemoryService userMemoryService) {
        this.userMemoryService = userMemoryService;
    }

    @Override
    public Mono<String> onSystemPrompt(
            Agent agent,
            RuntimeContext ctx,
            String sysPrompt,
            Function<String, Mono<String>> next) {

        String userId = ctx.getUserId();
        if (userId == null || userId.isBlank()) {
            return next.apply(sysPrompt);  // 无 userId 时不注入记忆
        }

        // 每次推理前动态加载最新记忆
        String memory = userMemoryService.loadUserMemory(userId);
        if (memory == null || memory.isBlank()) {
            return next.apply(sysPrompt);
        }

        String enrichedPrompt = sysPrompt + "\n\n" + memory;
        log.debug("[MemoryInject] userId={}, memorySize={}", userId, memory.length());
        return next.apply(enrichedPrompt);
    }
}
```

**注册方式**（在 `AgentFactory.buildAgent()` 中）：

```java
// AgentFactory 注入 UserMemoryService
private final UserMemoryService userMemoryService;

return HarnessAgent.builder()
    // ...
    .middleware(new TokenMonitorMiddleware())
    .middleware(new CostControlMiddleware(costTracker))
    .middleware(new MemoryInjectMiddleware(userMemoryService))  // ← 每次推理前动态注入
    .build();
```

**Middleware 链执行顺序**：
```
请求进入 → TokenMonitorMiddleware.onReasoning
         → CostControlMiddleware.onModelCall
         → MemoryInjectMiddleware.onSystemPrompt  ← 在此注入用户记忆
         → CompactionMiddleware（上下文压缩）
         → LLM 推理
```
