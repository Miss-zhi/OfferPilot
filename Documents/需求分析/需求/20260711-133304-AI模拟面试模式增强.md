# AI 模拟面试模式增强

## 1. 背景与目标

### 1.1 背景

需求规格说明书（§71-83）定义了 4 种面试模式（技术深挖/行为面试/系统设计/压力面试），但当前代码存在以下问题：

- `InterviewSession` 已有 `interviewMode` 字段，但 `MockInterviewTool.generate_next_question()` **完全不使用**它
- `generate_next_question` 的 `determineCategory()` 和 `determineDifficulty()` 仅基于 context 关键词和固定 PHASE_SEQUENCE 轮转
- 需求要求的"压力面试追问质疑"没有结构性支持，`analyze_answer` 不返回追问指导
- 需求要求的"根据简历项目经历深挖技术细节"没有工具支持

### 1.2 目标

1. `generate_next_question` 读取 `interviewMode` 字段，4 种模式产生不同的出题策略
2. 新增"压力追问"逻辑：当处于压力模式时，`analyze_answer` 附加追问指导
3. 新增 `resume_deep_dive` 参数，支持根据简历项目经历生成深挖题
4. 重构 PHASE_SEQUENCE 为模式感知的阶段轮转

## 2. 功能清单

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| 模式切换 | interviewMode 驱动 4 种出题策略 | P0 | TECH_DEEP / BEHAVIOR / SYSTEM_DESIGN / PRESSURE |
| 模式切换 | 模式感知的阶段轮转 | P0 | 不同模式的 phase 序列和难度递进不同 |
| 技术深挖 | 根据简历项目经历追问 | P0 | 接受 `resume_text` 参数，提取项目关键词生成深挖题 |
| 行为面试 | STAR 类问题倾向 | P1 | 增加行为面试阶段权重，偏向"说一个...经历"类问题 |
| 系统设计 | 开放型设计题 | P1 | 提供"设计一个...系统"类题目，不限具体答案 |
| 压力面试 | 追问质疑机制 | P0 | 回答后 LLM 基于指导进行追问，模拟压力场景 |
| 压力面试 | 难度激进上调 | P1 | 压力模式下难度至少 medium，均分 >60 直接 hard |

## 4. 技术方案

### 4.1 整体架构

```
用户选择模式 + 上传简历（可选）
    ↓
主Agent → spawn mock_interviewer(sessionId, mode="TECH_DEEP")
    ↓
每轮循环：
  ├── generate_next_question(context, mode, resume_text)
  │     ├── 读取 session.interviewMode → 选择策略
  │     ├── 技术深挖模式：resume_text 非空时提取项目关键词
  │     ├── 系统设计模式：提供"设计xxx系统"类指导
  │     └── 压力面试模式：输出出题指导 + 预期追问方向
  ↓
  ├── LLM 根据指导生成题目
  ↓
  ├── 用户回答
  ↓
  └── analyze_answer(question, answer, mode)
        ├── 正常模式：返回 3 维评估指导
        └── 压力模式：额外返回追问指导（followUpGuidance）
```

### 4.2 MockInterviewTool 重构

```java
@Tool(name = "generate_next_question",
      description = "获取下一个面试题的出题指导（含岗位、阶段、难度、历史题目、面试模式），由你据此生成并提问")
public NextQuestionResult generateNextQuestion(
        @ToolParam(name = "context", description = "当前面试上下文") String context,
        @ToolParam(name = "mode", description = "面试模式：TECH_DEEP/BEHAVIOR/SYSTEM_DESIGN/PRESSURE，不传则默认为TECH_DEEP", required = false) String mode,
        @ToolParam(name = "resume_text", description = "简历文本（技术深挖模式时用于提取项目关键词）", required = false) String resumeText) {

    log.info("generate_next_question: mode={}, hasResume={}", mode, resumeText != null);

    String sessionId = parseSessionId(context);
    String effectiveMode = resolveMode(mode, sessionId);
    int questionNumber = countPreviousQuestions(context) + 1;

    // 读取 session.role、session.targetCompany 等上下文
    String role = extractRole(context, sessionId);

    // 模式感知的出题策略
    String category = determineCategoryByMode(effectiveMode, context, questionNumber);
    String difficulty = determineDifficultyByMode(effectiveMode, context, sessionId, questionNumber);
    Double avgScore = fetchAverageScore(sessionId);
    List<String> prevQuestions = fetchPreviousQuestions(sessionId);

    // 技术深挖：提取简历项目关键词
    List<String> projectKeywords = List.of();
    if ("TECH_DEEP".equals(effectiveMode) && resumeText != null && !resumeText.isBlank()) {
        projectKeywords = extractProjectKeywords(resumeText);
    }

    String guidance = buildGuidanceByMode(effectiveMode, role, category, difficulty,
            questionNumber, avgScore, projectKeywords);

    persistQuestion(sessionId, guidance, category, difficulty);
    return new NextQuestionResult(guidance, category, difficulty, role,
            questionNumber, avgScore, prevQuestions, effectiveMode);
}
```

### 4.3 四种模式策略（InterviewModeService）

```java
@Service
public class InterviewModeService {

    /** 各模式下的阶段序列 */
    public List<String> getPhaseSequence(String mode) {
        return switch (mode) {
            case "TECH_DEEP" -> List.of(
                "自我介绍", "基础知识", "项目经验", "项目经验",
                "系统设计", "项目经验", "底层原理", "项目经验",
                "架构设计", "性能优化", "项目经验", "职业规划");
            case "BEHAVIOR" -> List.of(
                "自我介绍", "项目经验", "行为面试", "行为面试",
                "项目经验", "情景分析", "行为面试", "压力测试",
                "项目经验", "行为面试", "情景分析", "职业规划");
            case "SYSTEM_DESIGN" -> List.of(
                "自我介绍", "基础知识", "系统设计", "系统设计",
                "基础知识", "系统设计", "架构设计", "系统设计",
                "性能优化", "系统设计", "架构设计", "职业规划");
            case "PRESSURE" -> List.of(
                "自我介绍", "专业技能", "专业技能", "项目经验",
                "压力测试", "情景分析", "压力测试", "项目经验",
                "压力测试", "专业知识", "压力测试", "行为面试");
            default -> PHASE_SEQUENCE;
        };
    }

    /** 模式感知的难度递进 */
    public String determineDifficulty(String mode, Double avgScore, int questionNumber) {
        return switch (mode) {
            case "PRESSURE" -> {
                if (avgScore != null && !avgScore.isNaN() && avgScore >= 60) yield "hard";
                if (questionNumber > 3) yield "hard";
                yield "medium";  // 压力模式起步 medium
            }
            case "SYSTEM_DESIGN" -> {
                if (questionNumber <= 2) yield "easy";
                if (questionNumber <= 5) yield "medium";
                yield "hard";
            }
            default -> {
                if (avgScore != null && !avgScore.isNaN()) {
                    if (avgScore >= 75) yield "hard";
                    if (avgScore >= 50) yield "medium";
                    yield "easy";
                }
                if (questionNumber > 5) yield "hard";
                if (questionNumber > 2) yield "medium";
                yield "easy";
            }
        };
    }

    /** 从简历文本提取项目关键词 */
    public List<String> extractProjectKeywords(String resumeText) {
        // 提取项目经历段落原文，不做技术关键词判断。
        // 技术关键词识别交给 LLM，避免硬编码技术栈列表导致仅适配程序员。
        List<String> projectFragments = new ArrayList<>();
        for (String line : resumeText.split("\n")) {
            if (line.contains("项目") || line.contains("实习") || line.contains("开源")
                    || line.contains("负责") || line.contains("参与")) {
                projectFragments.add(line.trim());
            }
        }
        return projectFragments.stream().distinct().limit(10).toList();
    }
}
```

### 4.4 出题指导构建（按模式）

```java
private String buildGuidanceByMode(String mode, String role, String category,
        String difficulty, int questionNumber, Double avgScore,
        List<String> projectKeywords) {

    StringBuilder sb = new StringBuilder();
    sb.append("请为").append(role).append("岗位生成一道").append(category)
      .append("类面试题，难度").append(difficulty).append("，第").append(questionNumber).append("题。");

    // 模式特定指令
    switch (mode) {
        case "TECH_DEEP" -> {
            if (!projectKeywords.isEmpty()) {
                sb.append("候选人简历涉及以下项目经历，请从中识别关键技术栈并深挖：\n")
                  .append(String.join("\n", projectKeywords))
                  .append("\n请围绕其中一个技术点深挖，要求候选人从原理、实践、优化三个层次回答。");
            } else {
                sb.append("请深入追问技术原理和底层实现，要求候选人解释'为什么'而非'是什么'。");
            }
        }
        case "BEHAVIOR" -> {
            sb.append("请用STAR框架提问，要求候选人描述具体情境-任务-行动-结果。");
        }
        case "SYSTEM_DESIGN" -> {
            sb.append("请给出一个开放型系统设计问题，要求候选人从需求分析→架构选型→详细设计逐步展开。");
        }
        case "PRESSURE" -> {
            sb.append("这是压力面试模式。请提出有挑战性的问题，并在候选人回答后质疑其方案的边界条件、替代方案和潜在缺陷。");
        }
    }

    if (avgScore != null && !avgScore.isNaN()) {
        sb.append("历史均分").append(String.format("%.0f", avgScore)).append("分。");
    }

    sb.append("直接输出题目文本，不要加前缀说明。");
    return sb.toString();
}
```

### 4.5 AnswerAnalyzeTool 压力追问增强

```java
// 在 buildGuidance 中新增压力模式追问指导
private String buildGuidance(String question, String answer, String mode) {
    String base = String.format("""
            请分析以下面试回答：...
            【面试问题】%s
            【用户回答】%s
            从三个维度评分...
            """, question, answer);

    if ("PRESSURE".equals(mode)) {
        base += """
            
            【压力追问指导】
            请额外生成 2-3 个追问问题，以测试候选人回答的深度和抗压能力：
            1. 追问边界条件：挑战方案的极限场景和边界情况
            2. 追问替代方案：为什么选择该方案而非其他可行方案
            3. 追问具体实现：要求细化某个技术决策的实现细节
            追问应针对候选人的具体回答内容，不要使用通用模板。
            """;
    }
    return base;
}
```

### 4.6 AgentFactory 变更

```java
private SubagentDeclaration buildMockAgent() {
    return SubagentDeclaration.builder()
        .name("mock_interviewer")
        .description("模拟面试官...")
        .inlineAgentsBody("""
            你是面试官。面试开始前，请先确认面试模式：
            - 技术深挖 (TECH_DEEP)：适合技术岗，深挖项目经历和底层原理
            - 行为面试 (BEHAVIOR)：评估软素质和沟通能力
            - 系统设计 (SYSTEM_DESIGN)：考察架构设计能力
            - 压力面试 (PRESSURE)：测试抗压能力和临场反应

            如果没有明确指定，根据用户岗位默认选择：
            - 后端/算法/数据 → TECH_DEEP
            - 产品/运营 → BEHAVIOR
            - 架构师/高级开发 → SYSTEM_DESIGN

            每轮流程：
            1. 调用 generate_next_question(mode, context, resume_text) 获取出题指导
            2. 用自然语言提问
            3. 用户回答后调用 analyze_answer 分析
            4. 压力模式下根据 followUpGuidance 追问
            5. 每 3-4 题给简短反馈
            6. 面试结束（5-8 题）输出总体评价""")
        .tools(List.of("generate_next_question", "search_answers", "analyze_answer"))
        .build();
}
```
