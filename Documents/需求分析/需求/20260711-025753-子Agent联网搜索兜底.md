# 子 Agent RAG 未命中时自动联网搜索兜底

## 1. 背景与目标

### 背景

OfferPilot 当前有 7 个子 Agent，其中 `company_researcher`、`study_planner`、`salary_advisor` 的工具白名单仅包含 RAG 类 `@Tool`（如 `search_company_interviews`、`search_questions`、`search_salary` 等）。这些工具检索链路为：

```
@Tool → KnowledgeBaseService → VectorSearchService → Milvus 向量检索
                                                  → MySQL LIKE 回退（仅部分工具）
```

当本地知识库未导入相关文档、或 `op_salary_record` 表无对应数据时，检索返回空结果。**子 Agent 无权调用 `web_search` MCP 工具**，导致无法联网兜底，用户得到"未找到相关信息"的空响应。

### 目标

为上述 3 个子 Agent 增加 `web_search` MCP 工具访问权限，并在 sysPrompt 中配置"RAG 未命中 → 触发联网搜索"的决策逻辑，使子 Agent 能在本地数据不足时自主从互联网获取信息。

### 核心术语

| 术语 | 说明 |
|------|------|
| RAG 检索 | 基于 Milvus 向量数据库 + Embedding 的本地知识库语义搜索 |
| MCP 联网搜索 | 通过 `open-websearch` Docker 容器（百度/Bing）进行的互联网搜索 |
| 工具白名单 | `SubagentDeclaration.tools(List.of(...))` 中声明的子 Agent 可用工具列表 |

## 2. 功能清单

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| company_researcher | RAG 未命中时自动调用 `web_search` 联网搜索公司面经 | P0 | 当前最常触发空结果的场景 |

| tech_evaluator | — | 不涉及 | 技术评估无需联网 |
| expression_evaluator | — | 不涉及 | 表达分析无需联网 |
| mock_interviewer | — | 不涉及 | 模拟面试无需联网 |

## 4. 技术方案

### 整体思路

利用 AgentScope 的 ReAct 循环：子 Agent 在 sysPrompt 引导下，优先调用 RAG 类工具 → 若返回空结果或结果不足 → 自动调用 `web_search` 联网补充 → 整合两者结果回复用户。

### 前置条件（已满足）

- `workspace/tools.json` 已配置 `web-search` MCP Server（`http://localhost:3000/mcp`，Bing 引擎）
- `docker-compose.yml` 已包含 `web-search` 容器
- `PermissionContextState` 中 `web_search` 已配置为 `ALLOW`
- `HarnessAgent` 构建时通过 `.workspace(Path.of("./workspace"))` 自动加载 MCP 工具

### 变更点

**仅修改 `AgentFactory.java`**，涉及 3 个 `SubagentDeclaration` 构建方法：

#### (1) `buildCompanyAgent()` — company_researcher

```java
// 工具白名单增加 web_search
.tools(List.of("search_company_interviews", "search_questions", "web_search"))

// sysPrompt 增加兜底逻辑
.inlineAgentsBody("""
        你是一个面试情报分析师。
        1. 调用 search_company_interviews 检索目标公司的面试情报
        2. 调用 search_questions 检索高频考点的具体题目
        3. 如果以上两个工具返回的结果不足或为空，
           调用 web_search 从互联网搜索最新的面试经验
        4. 整合成'面试情报卡'，标注数据来源（知识库/互联网）
        信息要准确，标注数据来源和时间。""")
```

#### (2) `buildStudyAgent()` — study_planner

```java
// 工具白名单增加 web_search
.tools(List.of("track_progress", "search_resources", "search_questions", "web_search"))

// sysPrompt 增加兜底逻辑
.inlineAgentsBody("""
        你是一个学习计划规划师。
        1. 调用 track_progress 查看用户的学习数据
        2. 识别反复出现的薄弱点
        3. 按优先级排序
        4. 调用 search_resources 匹配学习资源
        5. 如果知识库中学习资源不足，调用 web_search 从互联网搜索
           相关教程、文档和练习材料
        6. 生成周学习计划
        计划要实际可执行，每天 1-2 小时为宜。""")
```

#### (3) `buildSalaryAgent()` — salary_advisor

```java
// 工具白名单增加 web_search
.tools(List.of("search_salary", "compare_offers", "generate_negotiation_script", "web_search"))

// sysPrompt 增加兜底逻辑
.inlineAgentsBody("""
        你是一个资深 HR 和职业规划顾问，有 15 年招聘和薪资谈判经验。
        你的职责：
        1. 调用 search_salary 检索目标公司+岗位的薪资范围
        2. 如果本地薪资数据库无记录，调用 web_search 从互联网搜索
           该公司和岗位的最新薪资行情
        3. 调用 compare_offers 对比多个 offer 的综合待遇
        4. 调用 generate_negotiation_script 生成谈判话术
        5. 结合市场数据给出客观建议
        薪资信息敏感，回复要专业、客观、有数据支撑。""")
```

### 架构示意

```
用户: "字节跳动 Java 面试题"
  └─ 主Agent → spawn company_researcher
       └─ company_researcher (ReAct 循环)
            ├─ ① search_company_interviews("字节跳动") → Milvus 检索 → 空
            ├─ ② search_questions("Java")              → Milvus 检索 → 空
            ├─ ③ web_search("字节跳动 Java 面试题")      → 联网搜索 → ✅ 有结果
            └─ 整合回复用户
```
