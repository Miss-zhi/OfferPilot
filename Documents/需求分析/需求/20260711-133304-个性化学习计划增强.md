# 个性化学习计划增强

## 1. 背景与目标

### 1.1 背景

需求规格说明书（§104-117）要求系统基于面试诊断结果生成个性化学习计划，但当前实现存在以下关键缺失：

- **无优先级排序算法**：`track_progress` 只返回结构化数据，薄弱点排序完全靠 LLM 自由发挥，需求要求"高频考点 × 低掌握度 = 最优先"
- **无动态调整机制**：`track_progress` 只读不写，无法根据学习进度更新计划状态
- **无定时任务**：需求要求的"每周提醒和检查进度"没有任何 scheduler 实现
- ProgressTrackTool 的 `buildGuidance()` 拼接自然语言成品文本，属 P2 级工具越权问题

### 1.2 目标

1. 新增 `prioritize_weaknesses` 工具，实现高频考点 × 低掌握度的量化优先级排序
2. 新增 `update_study_plan` 工具，支持动态更新学习计划（完成/延期/新增任务）
3. 引入 Spring `@Scheduled` 定时任务，每周生成学习报告 + 推送提醒
4. 重构 `ProgressTrackTool.buildGuidance()` 为返回结构化数据 + LLM 指导模式

## 4. 技术方案

### 4.1 整体架构

```
用户请求学习计划 → 主Agent → spawn study_planner
    ↓
study_planner:
  ├── track_progress(user_id) → 获取面试次数/掌握度/计划完成度
  ├── prioritize_weaknesses(user_id) → 量化排序薄弱点
  ├── 对每个薄弱点 → search_resources(topic) → 匹配学习资源
  ├── 对每个薄弱点 → search_questions(keyword) → 匹配自测题
  └── web_search 兜底（资源不足时）
    ↓
输出：周学习计划（按优先级）+ 资源清单 + 自测题

定时任务（每周日 20:00）：
  WeeklyReportScheduler → 汇总本周进度 → 生成周报 → 推送提醒
```

### 4.2 PriorityRankTool（优先级排序）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class PriorityRankTool {

    private final KnowledgeMasteryRepository masteryRepo;
    private final InterviewQuestionRepository questionRepo;
    private final SearchAnalyticsService analyticsService;

    @Tool(name = "prioritize_weaknesses",
          description = "按高频考点 × 低掌握度对薄弱知识点进行量化优先级排序")
    public PriorityResult prioritize(
            @ToolParam(name = "user_id", description = "用户ID") String userId) {

        log.info("prioritize_weaknesses called: userId={}", userId);

        // 1. 获取用户各知识点掌握度
        List<KnowledgeMastery> masteries = masteryRepo.findByUserId(userId);

        // 2. 获取知识点考频（从面试题库统计各知识点出现次数）
        Map<String, Integer> frequencyMap = analyticsService.getTopicFrequency();

        // 3. 计算优先级分数
        List<PriorityResult.RankedItem> items = new ArrayList<>();
        for (KnowledgeMastery m : masteries) {
            String topic = m.getKnowledgePoint();
            int score = m.getScore() != null ? m.getScore() : 0;
            int frequency = frequencyMap.getOrDefault(topic, 1);  // 至少1避免零权重

            // 优先级 = 考频权重 × (100 - 掌握度)
            // 考频高且掌握度低 → 最优先
            int priority = frequency * (100 - score);

            PriorityResult.RankedItem item = new PriorityResult.RankedItem();
            item.setTopic(topic);
            item.setCurrentScore(score);
            item.setFrequency(frequency);
            item.setPriority(priority);
            item.setUrgency(priority >= 5000 ? "HIGH" : priority >= 2000 ? "MEDIUM" : "LOW");
            items.add(item);
        }

        // 按优先级降序
        items.sort((a, b) -> Integer.compare(b.getPriority(), a.getPriority()));

        // 4. 构建 LLM 指导
        String guidance = buildPriorityGuidance(items);

        log.info("prioritize_weaknesses result: topics={}, topPriority={}",
                items.size(), items.isEmpty() ? "N/A" : items.get(0).getTopic());
        return new PriorityResult(guidance, items);
    }

    private String buildPriorityGuidance(List<PriorityResult.RankedItem> items) {
        StringBuilder sb = new StringBuilder();
        sb.append("请根据以下优先级排序生成学习计划建议：\n");
        sb.append("排序规则：高频考点 × 低掌握度 = 最优先\n\n");
        sb.append("| 优先级 | 知识点 | 当前掌握度 | 考频 | 紧急度 |\n");
        sb.append("|--------|--------|-----------|------|--------|\n");
        int rank = 1;
        for (var item : items) {
            sb.append(String.format("| %d | %s | %d | %d | %s |\n",
                    rank++, item.getTopic(), item.getCurrentScore(),
                    item.getFrequency(), item.getUrgency()));
        }
        sb.append("\n请按优先级从高到低安排每日学习任务，每天1-2个知识点，1-2小时为宜。");
        return sb.toString();
    }
}
```

```java
/**
 * 根据最新面试数据刷新学习计划优先级。
 * 每次面试分析完成后自动调用。
 */
@Transactional
public void refreshPlanPriorities(String userId) {
    List<KnowledgeMastery> masteries = masteryRepo.findByUserId(userId);
    List<StudyPlan> activePlans = planRepo.findByUserIdAndStatus(userId, "ACTIVE");

    // 获取新的薄弱点
    List<String> newWeaknesses = identifyNewWeaknesses(masteries);

    // 对比现计划，补充新薄弱点任务
    List<StudyPlan> updatedPlans = new ArrayList<>();
    for (StudyPlan plan : activePlans) {
        // ... 追加新任务到 tasksJson ...
        plan.setTotalCount(plan.getTotalCount() + newWeaknesses.size());
        updatedPlans.add(plan);
    }
    if (!updatedPlans.isEmpty()) {
        planRepo.saveAll(updatedPlans);  // 批量保存，避免 N+1
    }
```

### 4.4 定时任务（每周学习报告）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class WeeklyReportScheduler {

    private final StudyPlanRepository planRepo;
    private final KnowledgeMasteryRepository masteryRepo;
    private final InterviewSessionRepository sessionRepo;

    /**
     * 每周日 20:00 生成学习报告。
     * 汇总本周学习数据，更新 StudyPlan 周进度。
     */
    @Scheduled(cron = "0 0 20 ? * SUN")
    public void generateWeeklyReport() {
        log.info("Weekly report generation started");

        // 1. 获取所有 ACTIVE 计划的用户
        LocalDate weekStart = LocalDate.now().with(DayOfWeek.MONDAY);
        LocalDate weekEnd = LocalDate.now().with(DayOfWeek.SUNDAY);
        List<StudyPlan> activePlans = planRepo.findByStatus("ACTIVE");

        Set<String> userIds = activePlans.stream()
                .map(StudyPlan::getUserId)
                .collect(Collectors.toSet());

        for (String userId : userIds) {
            try {
                generateUserReport(userId, weekStart, weekEnd);
            } catch (Exception e) {
                log.error("Failed to generate report for userId={}: {}", userId, e.getMessage());
            }
        }

        log.info("Weekly report generation completed: {} users", userIds.size());
    }

    private void generateUserReport(String userId, LocalDate weekStart, LocalDate weekEnd) {
        // 本周面试次数
        long weekInterviews = sessionRepo.countByUserIdAndStartedAtBetween(
                userId, weekStart.atStartOfDay(), weekEnd.atTime(23, 59, 59));

        // 本周完成学习任务数
        List<StudyPlan> plans = planRepo.findByUserIdAndStatus(userId, "ACTIVE");
        int weekCompleted = plans.stream()
                .mapToInt(p -> p.getCompletedCount() != null ? p.getCompletedCount() : 0)
                .sum();

        // 本周掌握度变化
        List<KnowledgeMastery> masteries = masteryRepo.findByUserId(userId);
        int improved = 0;
        int declined = 0;
        for (KnowledgeMastery m : masteries) {
            if (m.getPreviousScore() != null && m.getScore() != null) {
                if (m.getScore() > m.getPreviousScore()) improved++;
                else if (m.getScore() < m.getPreviousScore()) declined++;
            }
        }

        log.info("Weekly report: userId={}, interviews={}, completed={}, improved={}, declined={}",
                userId, weekInterviews, weekCompleted, improved, declined);

        // 生成报告（写入 op_analysis_report，reportType=WEEKLY_REPORT）
        // ... 持久化报告 ...
    }
}
```

## 6. 接口设计

无需新增 REST API。学习计划通过对话接口触发，进度查询使用现有 `ProgressController`。

> **数据边界说明**：`prioritize_weaknesses` 工具在 Agent 对话中由 LLM 编排调用，其排序结果通过对话 SSE 流返回用户。
> `ProgressController` 的 REST API 返回原始的进度数据（面试次数、掌握度、计划完成度），**不含**优先级排序。
> 如需在 REST API 中获取优先级排序，应单独新增 Service 层方法（复用 `LearningPlanService` 的排序逻辑），而非让 API 依赖 Agent 工具。
