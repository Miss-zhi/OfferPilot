# 简历智能诊断增强

## 1. 背景与目标

### 1.1 背景

当前 `ResumeService.evaluateResume()` 仅返回简历原文 + 3 个宽泛评估维度（内容完整性/表达质量/岗位匹配度）的 LLM 指导文本，与需求规格说明书（§42-55）定义的完整简历诊断能力存在显著差距：

- **缺失 JD 对比**：`evaluate_resume` 不接收目标岗位 JD 参数，无法计算关键词匹配度
- **缺失 STAR 法则检查**：无结构化检查项目描述是否遵循"情境-任务-行动-结果"
- **缺失 3 项具体检查**：未实现"技能罗列没有层次""项目描述只写技术栈没写成果""缺少量化数据"的专项检查
- **ResumeService 工具越权**：`extractSkills()` 使用硬编码正则匹配"Java、Python、Figma"等具体技术栈关键词，仅适配程序员（P0 级工具越权问题）

### 1.2 目标

1. `evaluate_resume` 工具支持可选的 JD 参数，实现结构化 JD-简历匹配度计算
2. 新增 `check_star` 工具，对每段项目/工作经历进行 STAR 四要素检测
3. 新增 `check_resume_quality` 工具，专项检查 3 类常见问题
4. 将 `ResumeService.extractSkills()` 从硬编码正则改为 LLM 指导模式
5. 在 `resume_coach` 子Agent 的 SysPrompt 中集成 RAG 检索优秀简历模板

### 4.2 JD 匹配度计算（matchJd 新增方法）

```java
// ResumeService.java 新增方法
public JdMatchResult matchJd(String resumeText, String jdText) {
    if (jdText == null || jdText.isBlank()) {
        return new JdMatchResult(null, "未提供JD，跳过匹配度计算。", 0, 0, List.of(), List.of());
    }

    // 1. 提取 JD 关键技能词
    Set<String> jdSkills = extractKeywords(jdText);
    // 2. 提取简历关键技能词
    Set<String> resumeSkills = extractKeywords(resumeText);
    // 3. 计算覆盖度
    Set<String> matched = new HashSet<>(jdSkills);
    matched.retainAll(resumeSkills);
    Set<String> missing = new HashSet<>(jdSkills);
    missing.removeAll(resumeSkills);

    double matchRate = jdSkills.isEmpty() ? 0 : (double) matched.size() / jdSkills.size() * 100;
    int score = (int) Math.round(matchRate);

    String guidance = String.format("""
            请根据以下 JD 匹配分析指导生成自然语言建议：
            - 要求技能总数：%d，简历匹配数：%d，匹配率：%.0f%%
            - 匹配技能：%s
            - 缺失技能：%s
            请评估简历对目标岗位的匹配程度并给出优化方向。""",
            jdSkills.size(), matched.size(), matchRate, matched, missing);

    return new JdMatchResult(guidance, score, matchRate, new ArrayList<>(matched), new ArrayList<>(missing));
}
```

### 4.3 STAR 检查（analyzeStar 新增方法）

```java
// ResumeService.java 新增方法
public StarCheckResult analyzeStar(String resumeText) {
    // 1. 分割各段经历（基于项目/工作/实习等关键词切分）
    List<String> experiences = splitExperiences(resumeText);
    List<StarCheckResult.StarItem> items = new ArrayList<>();

    for (int i = 0; i < experiences.size(); i++) {
        String exp = experiences.get(i);
        StarCheckResult.StarItem item = new StarCheckResult.StarItem();
        item.setIndex(i + 1);
        item.setContent(exp.substring(0, Math.min(200, exp.length())));  // 截取前200字
        items.add(item);
    }

    // 2. 构建 LLM 指导 —— 工具只做文本分段，STAR 判断完全交给 LLM
    String guidance = buildStarGuidance(items);
    return new StarCheckResult(guidance, items, items.size());
}

/**
 * 将经历文本分段，构建 STAR 检测指导。
 * 工具只负责"找经历"，不负责"判断 STAR"——判断是 LLM 的职责。
 */
private String buildStarGuidance(List<StarCheckResult.StarItem> items) {
    StringBuilder sb = new StringBuilder();
    sb.append("请对以下简历中的每段项目/工作经历进行 STAR 四要素完整性检测：\n");
    sb.append("- S (Situation 情境)：是否说明了项目背景、面临的挑战？\n");
    sb.append("- T (Task 任务)：是否明确了具体职责、目标或要求？\n");
    sb.append("- A (Action 行动)：是否描述了采用的技术方案、具体做法？\n");
    sb.append("- R (Result 结果)：是否提供了可量化的成果数据？\n\n");

    for (StarCheckResult.StarItem item : items) {
        sb.append(String.format("【经历 %d】%s\n", item.getIndex(), item.getContent()));
        sb.append("请标注：有哪些要素（S/T/A/R），缺失哪些要素，并为缺失要素给出修复建议。\n\n");
    }

    sb.append("请以表格形式输出每段经历的 STAR 完整性检查和改进建议。");
    return sb.toString();
}

/**
 * 基础文本分段 —— 按项目/工作/实习等关键词切分经历段落。
 * 这是纯粹的文本处理，不涉及语义判断。
 */
private List<String> splitExperiences(String text) {
    List<String> segments = new ArrayList<>();
    // 按常见分隔词切分：项目经历、工作经历、实习经历
    String[] parts = text.split("(?=(?:项目|工作|实习)(?:经历|经验|描述))");
    for (String part : parts) {
        String trimmed = part.trim();
        if (trimmed.length() > 20 && !trimmed.startsWith("技能") && !trimmed.startsWith("教育")) {
            segments.add(trimmed);
        }
    }
    return segments;
}
```

### 4.4 简历质量专项检查（checkQuality 新增方法）

```java
public QualityCheckResult checkQuality(String resumeText) {
    List<QualityCheckResult.RawData> rawData = new ArrayList<>();

    // 收集原始数据，不做正则判断 —— 判断交给 LLM
    // 数据1：技能相关行
    List<String> skillLines = extractSkillLines(resumeText);
    rawData.add(new QualityCheckResult.RawData("技能段落",
        String.join("\n", skillLines),
        skillLines.size() + " 行技能描述"));

    // 数据2：项目/工作经历行
    List<String> expLines = extractExperienceLines(resumeText);
    rawData.add(new QualityCheckResult.RawData("项目经历段落",
        String.join("\n", expLines),
        expLines.size() + " 段项目/工作经历"));

    // 数据3：项目经历中的数值统计（辅助基础统计）
    long quantifiedCount = expLines.stream()
        .filter(l -> l.matches(".*\\d+[%％倍万千亿kKwWmM].*")).count();
    int totalExpCount = Math.max(expLines.size(), 1);

    String guidance = buildQualityGuidance(rawData, quantifiedCount, totalExpCount);
    return new QualityCheckResult(guidance, rawData, 0);
}

/**
 * 构建三项质量检查的 LLM 指导。
 * 工具提供原始文本片段和统计数据，LLM 做出判断。
 */
private String buildQualityGuidance(List<QualityCheckResult.RawData> rawData,
        long quantifiedCount, int totalExpCount) {
    return String.format("""
            请对以下简历进行三项专项检查，逐项给出发现的问题和改进建议：

            【检查项 1：技能层次分类】
            检查技能是否按精通/熟悉/了解分类组织。
            技能段落：%s

            【检查项 2：量化数据覆盖度】
            统计：%d 段项目经历中，%d 段包含量化数字（占比 %.0f%%）。
            检查是否有足够的数据支撑成果描述。建议每段经历至少包含 1-2 个量化指标。
            项目经历：%s

            【检查项 3：技术栈罗列 vs 业务成果】
            检查是否存在"仅罗列技术名词但未说明业务价值"的描述。
            判断标准：描述中提到技术/工具时，是否同时说明了解决了什么业务问题、达成了什么效果。

            请以结构化格式输出：每项检查 → 是否通过 → 具体问题位置 → 改进建议。""",
            rawData.get(0).getContent(),
            totalExpCount, quantifiedCount,
            totalExpCount > 0 ? (double) quantifiedCount / totalExpCount * 100 : 0,
            rawData.get(1).getContent());
}

/** 提取技能相关行 —— 纯文本处理，不含语义判断 */
private List<String> extractSkillLines(String text) {
    List<String> lines = new ArrayList<>();
    boolean inSkillSection = false;
    for (String line : text.split("\n")) {
        if (line.contains("技能") || line.contains("技术栈") || line.contains("Skills")) {
            inSkillSection = true;
            continue;
        }
        if (inSkillSection && !line.trim().isEmpty() && line.trim().length() > 3) {
            lines.add(line.trim());
        }
        if (inSkillSection && line.trim().isEmpty()) {
            inSkillSection = false;
        }
    }
    return lines;
}

/** 提取经历相关行 —— 纯文本处理 */
private List<String> extractExperienceLines(String text) {
    List<String> lines = new ArrayList<>();
    boolean inExpSection = false;
    for (String line : text.split("\n")) {
        String trimmed = line.trim();
        if (trimmed.contains("项目") || trimmed.contains("工作") || trimmed.contains("实习")
                || trimmed.contains("Experience") || trimmed.contains("Project")) {
            inExpSection = true;
            continue;
        }
        if (inExpSection && !trimmed.isEmpty() && trimmed.length() > 10) {
            lines.add(trimmed);
        }
        if (inExpSection && trimmed.isEmpty()) {
            inExpSection = false;
        }
    }
    return lines;
}
```

### 4.5 ResumeService.extractSkills() 定位修正

**问题**：原实现使用硬编码正则 `Pattern.compile("(?:熟练掌握|精通|熟悉)(Java|Python|Figma)...")` 匹配具体技术栈关键词，仅适配程序员，违反"工具不替代 LLM 做语义判断"原则。

**修正方向**：`extractSkills()` 降级为纯文本分段辅助方法 —— 从简历中找出"技能"相关段落并返回原文，由 LLM 负责分类和解读。

```java
/**
 * 从简历文本中提取技能相关行（纯文本处理，不含语义判断）。
 * 职责：定位"技能"章节 → 返回原文片段。
 * 不负责：识别具体技能名称、判断技能类别、匹配岗位要求。
 */
private List<String> extractSkills(String text) {
    List<String> skillLines = new ArrayList<>();
    boolean inSkillSection = false;
    for (String line : text.split("\n")) {
        if (line.contains("技能") || line.contains("技术栈") || line.contains("Skills")) {
            inSkillSection = true;
            continue;
        }
        if (inSkillSection && !line.trim().isEmpty()
            && (line.contains("：") || line.contains(":") || line.contains("、"))) {
            skillLines.add(line.trim());
        }
        if (inSkillSection && line.trim().isEmpty()) {
            inSkillSection = false;
        }
    }
    return skillLines.stream().distinct().limit(10).toList();
}
```

> **设计说明**：此方法定位为"文本锚点定位器"——它帮 LLM 定位简历中哪里写了技能，但不替 LLM 判断这些技能是什么、水平如何、是否匹配 JD。技能识别和 JD 匹配由 LLM 基于 guidance 完成。

## 6. 接口设计

无需新增 REST API。简历诊断完全通过 `ChatController` 的对话接口触发，由 Agent 自动编排工具调用。

### 对话触发示例

```
POST /api/v1/offerpilot/chat/stream

{
  "message": "请帮我诊断简历，目标岗位是字节跳动后端开发，JD如下：\n负责高并发系统设计与开发，精通Java/Go，熟悉分布式系统原理...",
```
