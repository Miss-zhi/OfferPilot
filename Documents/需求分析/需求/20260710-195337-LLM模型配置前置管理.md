# LLM 模型配置前置管理

## 1. 背景与目标

### 背景

当前 OfferPilot 的 LLM 模型配置完全硬编码在 `application.yml` 中（provider、model-name、api-key 等），所有用户共享同一套模型参数。存在以下问题：

- **无法按用户切换模型**：所有用户的 Agent 会话使用同一个模型（如 `dashscope:qwen-max`）
- **管理员无法动态管控**：新增/切换模型需要修改配置文件并重启服务
- **用户无法自选**：不同用户对模型性价比、响应速度有不同偏好，但无选择空间

### 目标

将 LLM 模型配置能力**前置到前端管理界面**，实现：

1. 管理员在后台**管理模型池**（启用/禁用、设置全局默认）
2. 用户在前端个人设置中**选择默认模型**（从管理员启用的模型池中选取）
3. 用户**自定义私有模型**（自行输入 provider + API Key + 模型列表链接）
4. Agent 构建时**按用户偏好优先**选择模型，未设置时 fallback 到全局默认

### 术语表

| 术语 | 说明 |
|------|------|
| 模型池 | 管理员启用后可供所有用户选择的模型集合 |
| 全局默认模型 | 管理员设置的、新用户未配置偏好时的兜底模型 |
| 用户默认模型 | 用户在个人设置中选择的偏好模型，优先级高于全局默认 |
| 模型提供方 (provider) | LLM 服务商标识，如 `dashscope`、`openai` |
| 模型列表链接 | 调用 LLM 服务商 API 获取可用模型名称的 URL |
| 私有模型 | 用户自行配置 provider + API Key + 模型列表链接生成的个人模型 |

### 4.2 Agent 构建流程变更

`AgentFactory.buildAgent(userId)` 改造为：

```java
private HarnessAgent buildAgent(String userId) {
    // 1. 查询用户模型偏好
    ModelConfig modelConfig = userModelService.getUserModelConfig(userId);
    
    // 2. 默认模型最后兜底
    if (modelConfig == null) {
        modelConfig = modelConfigService.getGlobalDefault();
    }
    
    // 3. 若均未配置，使用 application.yml 的 agentscope.model.* 作为最终兜底
    String provider = modelConfig != null ? modelConfig.getProvider() : properties.getModel().getProvider();
    String modelName = modelConfig != null ? modelConfig.getModelName() : properties.getModel().getModelName();
    String apiKey = modelConfig != null ? modelConfig.getApiKey() : properties.getModel().getApiKey();
    
    String modelId = provider + ":" + modelName;
    
    // ... 后续构建逻辑 (toolkit, sysPrompt, middleware) 不变
}
```

### 4.3 模型名称拉取流程

管理员新增模型配置时：

```
1. 管理员选择 provider（从系统预设列表选取），输入 API Key
2. 系统自动填入该 provider 的默认 Base URL + 模型列表链接（管理员可修改）
3. 后端调用 {模型列表链接}，使用该 provider 对应的认证 Header 和 API 格式
4. 解析响应 JSON（按 OpenAI / Anthropic / Gemini 三种格式适配），提取模型名称列表
5. 将配置存入 op_model_config，模型名称存入 op_model_name
```

**系统预设 Provider 配置表**（见 7.2 节完整清单）：管理员选择 provider 后，Base URL、模型列表链接、认证 Header 类型均自动填充，无需手动输入。

**API 格式适配逻辑**（`ModelListFetcher` 服务）：

```java
// 按 provider 的 api_format 字段选择解析器
public List<String> fetchModelNames(ModelConfig config) {
    String url = config.getModelListUrl();
    String apiKey = decrypt(config.getApiKey());
    
    HttpResponse response = switch (config.getApiFormat()) {
        case "openai"   -> httpGet(url, Map.of("Authorization", "Bearer " + apiKey));
        case "anthropic"-> httpGet(url, Map.of("x-api-key", apiKey, "anthropic-version", "2023-06-01"));
        case "gemini"   -> httpGet(url, Map.of("x-goog-api-key", apiKey));
        default         -> httpGet(url, Map.of("Authorization", "Bearer " + apiKey));
    };
    
    return switch (config.getApiFormat()) {
        case "openai"   -> parseOpenAI(response);   // data[].id
        case "anthropic"-> parseAnthropic(response); // data[].id
        case "gemini"   -> parseGemini(response);    // models[].name → 去掉 "models/" 前缀
        default         -> parseOpenAI(response);
    };
}
```

## 5. 数据库变更

### 5.1 新建表 `op_model_config`

```sql
CREATE TABLE op_model_config (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    provider        VARCHAR(64)   NOT NULL COMMENT '模型提供方，如 dashscope / openai / anthropic',
    base_url        VARCHAR(256)  NOT NULL COMMENT 'API Base URL，如 https://api.openai.com/v1',
    api_key         VARCHAR(512)  NOT NULL COMMENT 'API Key（AES 加密存储）',
    api_format      VARCHAR(16)   NOT NULL DEFAULT 'openai' COMMENT 'API 格式: openai / anthropic / gemini',
    auth_header_type VARCHAR(16)  NOT NULL DEFAULT 'bearer' COMMENT '认证Header: bearer / x-api-key / x-goog-api-key / none',
    model_list_url  VARCHAR(512)  NOT NULL COMMENT '模型列表获取链接',
    default_model_name VARCHAR(128) COMMENT '该配置下的默认模型名称（管理员从拉取的列表中选一个）',
    is_enabled      TINYINT(1)    NOT NULL DEFAULT 1 COMMENT '是否启用',
    is_global_default TINYINT(1)  NOT NULL DEFAULT 0 COMMENT '是否为全局默认模型配置',
    created_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_by       VARCHAR(64)   DEFAULT NULL COMMENT '创建人',
    update_by       VARCHAR(64)   DEFAULT NULL COMMENT '更新人',
    INDEX idx_is_enabled (is_enabled),
    INDEX idx_is_global_default (is_global_default),
    INDEX idx_provider (provider)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='模型配置表';
```

### 5.2 新建表 `op_model_name`

```sql
CREATE TABLE op_model_name (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
    model_config_id BIGINT        NOT NULL COMMENT '关联 op_model_config.id',
    model_name      VARCHAR(128)  NOT NULL COMMENT '模型名称，如 qwen-max / gpt-4',
    is_available    TINYINT(1)    NOT NULL DEFAULT 1 COMMENT '管理员是否勾选为可用',
    created_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at      DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_model_config_id (model_config_id),
    CONSTRAINT fk_model_name_config FOREIGN KEY (model_config_id) REFERENCES op_model_config(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='模型名称表（从API拉取）';
```

### 5.3 修改表 `op_user`

```sql
ALTER TABLE op_user
    ADD COLUMN default_model_config_id BIGINT DEFAULT NULL COMMENT '用户默认模型配置ID，关联 op_model_config.id',
    ADD COLUMN private_model_config_id BIGINT DEFAULT NULL COMMENT '用户私有模型配置ID（若非空则优先使用）',
    ADD CONSTRAINT fk_user_model_config FOREIGN KEY (default_model_config_id) REFERENCES op_model_config(id) ON DELETE SET NULL,
    ADD CONSTRAINT fk_user_private_model FOREIGN KEY (private_model_config_id) REFERENCES op_model_config(id) ON DELETE SET NULL;
```

## 7. 补充说明

### 7.1 API Key 安全存储

- `api_key` 字段存储前使用 AES 加密（密钥从环境变量注入）
- 接口返回时脱敏显示（如 `sk-****-xxxx`）
- 仅 `AgentFactory.buildAgent()` 内部使用时解密

### 7.2 系统预设 Provider 完整配置清单

根据对主流 LLM 服务商的调研，系统预置 8 个 Provider 配置：

#### OpenAI 兼容阵营（5家 — 统一 `Authorization: Bearer` 认证 + `GET /models`）

| # | Provider Key | 显示名称 | 默认 Base URL | 模型列表链接 | API 格式 | Key 模板 | 代表模型 |
|---|-------------|---------|--------------|-------------|---------|---------|---------|
| 1 | `dashscope` | 阿里百炼 (DashScope) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `{baseUrl}/models` | openai | `sk-xxx` | qwen-max, qwen-plus, qwen-turbo |
| 2 | `openai` | OpenAI | `https://api.openai.com/v1` | `{baseUrl}/models` | openai | `sk-xxx` | gpt-4.1, gpt-4o, gpt-4-turbo |
| 3 | `deepseek` | DeepSeek | `https://api.deepseek.com` | `https://api.deepseek.com/models` | openai | `sk-xxx` | deepseek-chat, deepseek-reasoner |
| 4 | `siliconflow` | 硅基流动 (SiliconFlow) | `https://api.siliconflow.cn/v1` | `{baseUrl}/models` | openai | `sk-xxx` | deepseek-ai/DeepSeek-R1, Qwen/Qwen2.5 |
| 5 | `volcengine` | 火山引擎 (豆包) | `https://ark.cn-beijing.volces.com/api/v3` | `{baseUrl}/models` | openai | `apikey-xxx` | doubao-seed-1.8, ark-code-latest |

> ⚠️ **火山引擎特殊性**：豆包模型需先在火山方舟控制台创建推理接入点（Endpoint ID，格式 `ep-xxx`），模型 ID 与接入点绑定。管理员需先完成此步骤再配置。

#### 非 OpenAI 兼容阵营（3家 — 认证/响应格式各异）

| # | Provider Key | 显示名称 | 默认 Base URL | 模型列表链接 | API 格式 | 认证 Header | 响应字段 |
|---|-------------|---------|--------------|-------------|---------|-----------|---------|
| 6 | `anthropic` | Anthropic (Claude) | `https://api.anthropic.com` | `https://api.anthropic.com/v1/models` | anthropic | `x-api-key` + `anthropic-version: 2023-06-01` | `data[].id`, `data[].display_name` |
| 7 | `gemini` | Google Gemini | `https://generativelanguage.googleapis.com/v1beta` | `{baseUrl}/models` | gemini | `x-goog-api-key` | `models[].name`（格式 `models/gemini-xxx`，需去掉前缀） |
| 8 | `ollama` | Ollama (本地) | `http://localhost:11434/v1` | `{baseUrl}/models` | openai | 无需认证 | `data[].id` |

#### 三种 API 响应格式对比

**OpenAI 格式**（5家 + Ollama）：
```json
{"object":"list","data":[{"id":"qwen-max","object":"model","created":1686935002,"owned_by":"dashscope"}]}
```
→ 解析路径：`data[].id`

**Anthropic 格式**：
```json
{"data":[{"id":"claude-opus-4-6","display_name":"Claude Opus 4.6","created_at":"2026-02-04T00:00:00Z","type":"model"}],"has_more":true}
```
→ 解析路径：`data[].id`

**Gemini 格式**：
```json
{"models":[{"name":"models/gemini-2.5-pro","displayName":"gemini-2.5-pro","supportedActions":["generateContent"]}]}
```
→ 解析路径：`models[].name`，去掉 `"models/"` 前缀得到 `gemini-2.5-pro`

### 7.3 数据库字段补充

为支持 Provider 差异化配置，`op_model_config` 表需额外新增字段：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `base_url` | VARCHAR(256) | API Base URL | `https://api.openai.com/v1` |
| `api_format` | VARCHAR(16) | API 格式类型 | `openai` / `anthropic` / `gemini` |
| `auth_header_type` | VARCHAR(16) | 认证 Header 类型 | `bearer` / `x-api-key` / `x-goog-api-key` / `none` |

### 7.4 向后兼容

- 若 `op_model_config` 表为空（首次部署），Agent 构建 fallback 到 `application.yml` 中的 `agentscope.model.*` 配置
- 现有功能不受影响
- 新增的 Provider 预设配置以 Java Enum 形式维护在代码中（`ProviderPreset` 枚举），确保数据库配置与代码解耦
