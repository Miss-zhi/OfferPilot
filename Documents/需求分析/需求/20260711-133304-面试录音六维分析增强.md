# 面试录音六维分析增强

## 1. 背景与目标

### 1.1 背景

需求规格说明书（§85-102）定义面试分析的 6 个维度，但当前 `AnswerAnalyzeTool.analyze_answer()` 仅覆盖 3 个（专业深度/表达能力/内容覆盖度），欠缺 3 个独立维度：

| 需求维度 | 当前状态 | 差距 |
|---------|---------|------|
| 技术深度（RAG 对标） | ⚠️ 有但靠 LLM，无 RAG 对标机制 | `tech_evaluator` SysPrompt 提了对标但工具不执行 |
| 表达逻辑 | ✅ `analyze_answer` 有 | — |
| **知识盲区** | ❌ 完全缺失 | 需 RAG 检索标答 + 标记遗漏点 |
| **时间分配** | ❌ 完全缺失 | 需时间戳分析（回答过长/过短） |
| **自信度** | ❌ 完全缺失 | 需文本特征分析（口头禅/犹豫词） |
| 亮点时刻 | ⚠️ `analyze_answer` 有"亮点"字段 | 需求要求 RAG 对标优秀答案 |

### 1.2 目标

1. 新增 `detect_knowledge_gaps` 工具，RAG 检索正确答案并对比用户回答，标记遗漏知识点
2. 新增 `analyze_time_allocation` 工具，根据回答字符数 / 语速估算判断时长合理性
3. 新增 `analyze_confidence` 工具，文本特征分析（口头禅密度、犹豫词频率、自我修正模式）
4. 增强 `tech_evaluator` 子Agent，将 RAG 对标从 SysPrompt 建议改为工具保障

## 2. 功能清单

| 功能模块 | 功能点 | 优先级 | 说明 |
|---------|--------|--------|------|
| 知识盲区 | RAG 检索标答对标 | P0 | Milvus 向量检索标准答案，词法对比标记遗漏点 |
| 知识盲区 | 遗漏知识点标注与补充建议 | P0 | 输出遗漏点 + 建议学习资源 |
| 时间分配 | 回答时长合理性判断 | P0 | 计算字符数估算时长，对比建议范围 |
| 时间分配 | 整体时间分布图 | P1 | 每题时间占比 + 超时/过短警告 |
| 自信度 | 口头禅密度统计 | P0 | 统计"嗯/那个/就是/然后/这个"等停顿填充词 |
| 自信度 | 犹豫/自我修正模式检测 | P1 | 检测"...""我认为...不是，实际上..."等修正模式 |
| 自信度 | 自信度综合评分 | P0 | 口头禅密度 + 修正次数 → 0-100 自信度分数 |

## 4. 技术方案

### 4.1 整体架构

```
用户上传录音 / 粘贴文字 → 主Agent
    ↓
并行 spawn：
  ├── tech_evaluator:
  │     ├── search_answers(问题) → RAG 检索标答
  │     ├── detect_knowledge_gaps(用户回答, 标准答案) → 标记遗漏点
  │     └── analyze_answer(问题, 回答) → 3 维核心评分
  │
  └── expression_evaluator:
        ├── analyze_answer(问题, 回答) → 表达逻辑评分
        ├── analyze_time_allocation(回答) → 时长分析
        └── analyze_confidence(回答) → 自信度分析
    ↓
主Agent 整合 → 6 维分析报告
```

### 4.2 KnowledgeGapTool（知识盲区检测）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class KnowledgeGapTool {

    private final KnowledgeBaseService kbService;

    @Tool(name = "detect_knowledge_gaps",
          description = "RAG检索标准答案并对比用户回答，标记遗漏的知识点和概念")
    public KnowledgeGapResult detectGaps(
            @ToolParam(name = "question", description = "面试问题") String question,
            @ToolParam(name = "user_answer", description = "用户回答文本") String userAnswer,
            @ToolParam(name = "top_k", description = "检索标答数量", required = false) Integer topK) {

        log.info("detect_knowledge_gaps called: questionLen={}, answerLen={}",
                question != null ? question.length() : 0,
                userAnswer != null ? userAnswer.length() : 0);

        // 1. RAG 检索标准答案
        AnswerSearchResult standardAnswers = kbService.searchAnswers(
                question, topK != null ? topK : 3);

        // 2. 提取标答中的关键知识点/概念
        Set<String> standardConcepts = extractConcepts(standardAnswers);

        // 3. 提取用户回答中的知识点/概念
        Set<String> userConcepts = extractConceptsFromText(userAnswer);

        // 4. 计算遗漏
        Set<String> missing = new HashSet<>(standardConcepts);
        missing.removeAll(userConcepts);

        Set<String> covered = new HashSet<>(standardConcepts);
        covered.retainAll(userConcepts);

        double coverageRate = standardConcepts.isEmpty() ? 0
                : (double) covered.size() / standardConcepts.size() * 100;

        String guidance = buildGapGuidance(covered, missing, coverageRate);
        return new KnowledgeGapResult(guidance, new ArrayList<>(covered),
                new ArrayList<>(missing), Math.round(coverageRate));
    }
    
    /** 从标答中提取关键概念（分词 + 停用词过滤 + 专有名词匹配） */
    private Set<String> extractConcepts(AnswerSearchResult answers) {
        Set<String> concepts = new LinkedHashSet<>();
        for (var ans : answers.getAnswers()) {
            String text = ans.getContent();
            // 提取引号内术语、"xxx原理/定律/模式/算法/协议"等模式
            Pattern termPattern = Pattern.compile(
                "(?:["'"])([\\u4e00-\\u9fa5a-zA-Z]{2,20})(?:["'"])"
                + "|([\\u4e00-\\u9fa5a-zA-Z]+(?:原理|模式|算法|协议|定律|架构|模型|思想))");
            Matcher m = termPattern.matcher(text);
            while (m.find()) {
                String term = m.group(1) != null ? m.group(1) : m.group(2);
                concepts.add(term);
            }
        }
        return concepts;
    }
    
    /**
     * 构建知识盲区检测的 LLM 指导。
     * 词法对比结果作为辅助指标，但告知 LLM 需要做语义级别判断：
     * 用户可能用自己的话表达了标准答案中的概念（同义表达），词法匹配会漏判。
     */
    private String buildGapGuidance(Set<String> covered, Set<String> missing, double coverageRate) {
        return String.format("""
                请基于以下词法对比数据进行知识盲区分析：
                - 词法覆盖率：%.0f%%
                - 词法匹配到的知识点：%s
                - 词法遗漏的知识点：%s
    
                注意：词法对比仅检测表面术语匹配，你需要从语义层面复核：
                1. 遗漏点中是否有被用户用不同措辞覆盖的内容？
                2. 匹配点中是否有仅提及名称但未深入的内容？
                3. 最终确定真正的知识盲区并给出补充学习建议。""",
                coverageRate, covered, missing);
    }

    private Set<String> extractConceptsFromText(String text) {
        // 同上逻辑，从用户回答文本中提取
        return extractConcepts(new AnswerSearchResult(/* wrap text */));
    }
}
```

### 4.3 TimeAllocationTool（时间分配分析）

```java
@Slf4j
@Component
public class TimeAllocationTool {

    /** 中文正常语速约 250 字/分钟 */
    private static final int CHARS_PER_MINUTE = 250;
    /** 合理回答时长范围（秒） */
    private static final int MIN_SECONDS = 30;
    private static final int MAX_SECONDS = 180;
    private static final int IDEAL_MIN = 60;
    private static final int IDEAL_MAX = 120;

    @Tool(name = "analyze_time_allocation",
          description = "根据回答文本长度估算回答时长，判断是否合理（太短/太长/适中）")
    public TimeAllocationResult analyze(
            @ToolParam(name = "answers", description = "各题回答文本的JSON数组，每项{question, answer}") String answersJson) {

        log.info("analyze_time_allocation called");

        // 解析 JSON 数组
        List<QAPair> qaList = parseQAList(answersJson);
        List<TimeAllocationResult.TimeItem> items = new ArrayList<>();

        for (QAPair qa : qaList) {
            int chars = qa.answer.replaceAll("\\s+", "").length();
            int estimatedSeconds = chars * 60 / CHARS_PER_MINUTE;

            TimeAllocationResult.TimeItem item = new TimeAllocationResult.TimeItem();
            item.setQuestion(qa.question.substring(0, Math.min(50, qa.question.length())));
            item.setCharCount(chars);
            item.setEstimatedSeconds(estimatedSeconds);

            // 仅返回评估枚举值，不输出自然语言评语（评语由 LLM 生成）
            if (estimatedSeconds < MIN_SECONDS) {
                item.setAssessment("TOO_SHORT");
            } else if (estimatedSeconds > MAX_SECONDS) {
                item.setAssessment("TOO_LONG");
            } else if (estimatedSeconds < IDEAL_MIN) {
                item.setAssessment("ACCEPTABLE");
            } else if (estimatedSeconds <= IDEAL_MAX) {
                item.setAssessment("GOOD");
            } else {
                item.setAssessment("ACCEPTABLE");
            }
            items.add(item);
        }

        // 整体统计
        int totalSeconds = items.stream().mapToInt(TimeAllocationResult.TimeItem::getEstimatedSeconds).sum();
        long tooShort = items.stream().filter(i -> "TOO_SHORT".equals(i.getAssessment())).count();
        long tooLong = items.stream().filter(i -> "TOO_LONG".equals(i.getAssessment())).count();

        String guidance = String.format("""
                请根据以下时间分配数据生成自然语言分析：
                总计 %d 秒，%d 题过短（TOO_SHORT），%d 题过长（TOO_LONG）
                评估标准：GOOD=时长适中，ACCEPTABLE=可接受，TOO_SHORT=回答过短缺乏深度，TOO_LONG=回答过长需收敛
                请为每道题生成具体的改进建议。""",
                totalSeconds, tooShort, tooLong);

        return new TimeAllocationResult(guidance, items, totalSeconds, (int) tooShort, (int) tooLong);
    }
}
```

### 4.4 ConfidenceTool（自信度分析）

```java
@Slf4j
@Component
public class ConfidenceTool {

    /** 常见犹豫/填充词 */
    private static final Set<String> HESITATION_WORDS = Set.of(
            "嗯", "呃", "那个", "就是", "然后", "这个",
            "怎么说呢", "emmm", "啊", "哦");

    /** 自我修正模式 */
    private static final Pattern SELF_CORRECTION = Pattern.compile(
            "不[,，]?(?:是|对|应该|准确[的来说])|其实|我觉得|可能|大概|也许|似乎|好像");

    @Tool(name = "analyze_confidence",
          description = "分析回答文本中的口头禅密度、犹豫词频率和自我修正模式，评估自信心水平")
    public ConfidenceResult analyze(
            @ToolParam(name = "text", description = "完整的用户回答文本") String text) {

        log.info("analyze_confidence called: textLen={}", text != null ? text.length() : 0);

        if (text == null || text.isBlank()) {
            return new ConfidenceResult("文本为空，无法分析", 0, Map.of(), 0, 0);
        }

        // 总字数（去除空白）
        String clean = text.replaceAll("\\s+", "");
        int totalChars = clean.length();

        // 1. 口头禅统计
        Map<String, Integer> fillerCounts = new LinkedHashMap<>();
        for (String word : HESITATION_WORDS) {
            int count = countOccurrences(text, word);
            if (count > 0) {
                fillerCounts.put(word, count);
            }
        }
        int totalFillers = fillerCounts.values().stream().mapToInt(Integer::intValue).sum();
        double fillerDensity = totalChars > 0 ? (double) totalFillers / totalChars * 1000 : 0;

        // 2. 自我修正统计
        Matcher matcher = SELF_CORRECTION.matcher(text);
        int correctionCount = 0;
        while (matcher.find()) {
            correctionCount++;
        }
        double correctionDensity = totalChars > 0 ? (double) correctionCount / totalChars * 1000 : 0;

        // 3. 计算自信度评分（0-100）
        // 口头禅密度低 + 自我修正少 = 自信度高
        int confidenceScore = calculateConfidenceScore(fillerDensity, correctionDensity, totalChars);

        String assessment = confidenceScore >= 80 ? "自信" : confidenceScore >= 60 ? "适度" : "偏弱";

        String guidance = String.format("""
                请根据以下自信度分析数据生成自然语言评语：
                - 自信度评分：%d/100（%s）
                - 口头禅密度：%.1f 次/千字
                - 高频口头禅：%s
                - 自我修正次数：%d 次
                请给出提升表达自信度的具体建议。""",
                confidenceScore, assessment, fillerDensity, fillerCounts, correctionCount);

        return new ConfidenceResult(guidance, confidenceScore, fillerCounts,
                Math.round(fillerDensity * 10) / 10.0, correctionCount);
    }

    private int countOccurrences(String text, String word) {
        int count = 0;
        int idx = 0;
        while ((idx = text.indexOf(word, idx)) != -1) {
            count++;
            idx += word.length();
        }
        return count;
    }

    private int calculateConfidenceScore(double fillerDensity, double correctionDensity, int totalChars) {
        // 基础分 100，扣分规则（TODO: 系数待通过 A/B 测试校准）：
        // 口头禅密度 > 5/千字 扣分
        // 自我修正 > 2/千字 扣分
        // 文本过短额外扣分（样本不足）
        int score = 100;
        score -= (int) Math.min(40, fillerDensity * 5);   // TODO: 待校准
        score -= (int) Math.min(30, correctionDensity * 10); // TODO: 待校准
        if (totalChars < 50) {
            score -= 20;  // 样本不足
        }
        return Math.max(0, Math.min(100, score));
    }
}
```
