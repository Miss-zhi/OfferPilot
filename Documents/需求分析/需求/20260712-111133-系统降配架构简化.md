# OfferPilot 系统降配 — 架构简化方案

> **文档版本**：v1.0  
> **创建时间**：2026-07-12  
> **来源**：用户需求"系统降配，只保留三大核心功能，单 Agent 架构"  
> **关联文档**：`Documents/01-需求规格说明书.md`（功能 1/3/4）

---

## 1. 背景与目标

### 1.1 背景

当前 OfferPilot 已实现 1 主 Agent + 8 子 Agent 的多智能体协作架构，覆盖 6 大功能模块（简历诊断、公司情报、模拟面试、录音分析、学习计划、薪资谈判），共 18 个 @Tool 文件（13 个已注册 + 5 个未注册）+ 1 个 MCP 工具，10 个 REST Controller，19 张数据库表。

系统复杂度高、维护成本大，需要精简为核心功能集，降低部署和运维负担。

### 1.2 目标

1. **功能精简**：仅保留三大核心功能 — 简历智能诊断、AI 模拟面试、面试录音/文字分析
2. **架构简化**：取消全部 8 个子 Agent，改为单一主 Agent 直接调用工具
3. **代码瘦身**：移除不再需要的 Controller、Tool、Entity、Repository、Service
4. **数据库瘦身**：删除不再使用的表

### 1.3 核心术语

| 术语 | 说明 |
|------|------|
| 主 Agent | 唯一的 HarnessAgent 实例，直接调用所有工具，不再通过 spawn 派发子 Agent |
| 简历智能诊断 | 上传 PDF → 解析结构 → 多维度评估 → 输出优化建议 |
| AI 模拟面试 | 4 种模式（TECH_DEEP / BEHAVIOR / SYSTEM_DESIGN / PRESSURE），多轮对话，SSE 流式输出 |
| 面试录音/文字分析 | 支持上传音频（ASR 转写）+ 粘贴文本，6 维度深度分析 |
| 知识库 | Milvus 向量库 + RAG 检索，支撑简历对标、出题检索、答案对标 |
| MCP 联网搜索 | 通过 open-websearch 服务（Bing 引擎）兜底，知识库无命中时自动联网 |

## 4. 技术方案

### 4.1 整体架构变更

```
变更前（多 Agent 调度）：                  变更后（单一 Agent）：
                          
  ┌──────────────┐                        ┌──────────────────┐
  │  主 Agent     │                        │   OfferPilot      │
  │  (调度中心)    │                        │   全能助手         │
  └───┬───┬───┬──┘                        │                   │
      │   │   │  spawn/resume              │  直接调用所有工具    │
  ┌───┘   │   └───┐                        └──┬──┬──┬──┬──┬──┘
  ▼       ▼       ▼                          ▼  ▼  ▼  ▼  ▼  ▼
┌────┐ ┌────┐ ┌────┐                      parse  evaluate  generate
│子A │ │子B │ │子C │ ...                   resume resume  next
└──┬─┘ └──┬─┘ └──┬─┘                      question
   │       │       │                          
   ▼       ▼       ▼                      analyze  transcribe  search
 Tool    Tool    Tool                      answer   audio       questions
```

#### 4.2.1 移除的内容

#### 4.2.2 System Prompt 重构

```java
// 变更前：调度中心角色，禁止直接调用业务工具
// 变更后：全能助手角色，直接调用所有工具

private String buildSystemPrompt() {
    return """
            You are OfferPilot 面试诊断助手。

            【你的能力】
            1. 简历智能诊断 — 解析 PDF 简历，多维度评估，给出优化建议
            2. AI 模拟面试 — 扮演面试官，支持 TECH_DEEP/BEHAVIOR/SYSTEM_DESIGN/PRESSURE 模式
            3. 录音/文字分析 — 转写录音，六维深度分析面试表现

            【可用工具及使用场景】
            | 工具 | 使用场景 |
            | parse_resume | 用户上传 PDF 简历时解析 |
            | evaluate_resume | 简历解析后，评估质量并给出建议 |
            | generate_next_question | 模拟面试中，获取出题指导 |
            | analyze_answer | 用户回答后，分析回答质量；面试复盘分析 |
            | transcribe_audio | 用户上传录音文件时转写 |
            | search_questions | 检索面试题库 |
            | search_answers | 检索优秀答案对标 |
            | smart_search | 通用知识库语义检索 |
            | search_resources | 检索学习资源和教程 |
            | search | 知识库无结果时联网搜索兜底 |

            【面试模式说明】
            - TECH_DEEP: 深挖简历技术栈，追问底层原理
            - BEHAVIOR: 行为面试，STAR 法则评估软素质
            - SYSTEM_DESIGN: 系统设计，考察架构能力
            - PRESSURE: 压力面试，追问质疑，测试抗压能力

            【重要规则】
            1. 工具返回的是结构化指导，你需要基于指导生成自然语言回复
            2. 面试中每 3-4 题给简短反馈，结束时（5-8 题）输出总体评价
            3. 分析结果要客观、具体、可操作
            4. 知识库检索优先，无结果时自动联网兜底
            5. 绝不暴露内部工具名称和调用过程给用户
            """.stripIndent();
}
```

#### 4.2.3 Toolkit 精简

```java
// 保留 4 个工具分组 + 9 个工具 + MCP search
// 分组 1: knowledge_retrieval — search_questions, search_answers, smart_search, search_resources
// 分组 2: resume_analysis      — parse_resume, evaluate_resume
```

### 4.3 ChatController 变更

移除子 Agent 流式事件处理的特殊逻辑（`AGENT_END` 的 source 标签判断），简化 SSE 事件处理流程。主 Agent 直接返回结果，不需要区分父子 Agent 事件。

### 4.4 前端变更要点

1. **导航菜单精简**：移除"学习进度""薪资谈判"入口
2. **路由清理**：移除 /progress、/salary 路由（对应 App.tsx 中 ProgressPage、SalaryPage）
3. **前端文件删除**：
   - 页面：`web/src/ui/pages/progress/`、`web/src/ui/pages/salary/`
   - Service：`web/src/service/progressService.ts`、`web/src/service/salaryService.ts`
4. **保留文件不受影响**：resumeService.ts、reportService.ts、chatService.ts、kbService.ts、modelConfigService.ts、authService.ts

### 4.5 工具-LLM 职责分离

继续遵循现有架构范式：工具返回结构化指导数据，LLM 基于指导生成自然语言内容。

| 工具 | 返回内容 | LLM 职责 |
|------|---------|---------|
| evaluate_resume | 简历文本 + 评估指导 | 生成个性化优化建议 |
| generate_next_question | category/difficulty/guidance | 基于指导生成面试题 |
| analyze_answer | Q&A 对 + 评分指导 | 生成自然语言分析和反馈 |
| transcribe_audio | 转写文本 | 基于文本进行六维分析 |

### 6.3 Chat 流式接口（内部行为变化 + 新增 mode 参数）

```
POST /api/v1/offerpilot/chat/stream
```

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| sessionId | String | 否 | 会话 ID，新会话自动生成 |
| message | String | 是 | 用户消息文本 |
| mode | String | 否 | **新增字段**，面试模式：TECH_DEEP / BEHAVIOR / SYSTEM_DESIGN / PRESSURE（仅面试场景）。需同步修改 ChatRequest.java 新增 mode 字段 |

**响应：** SSE 事件流（text/event-stream）

```
data: {"type":"text","content":"好的，让我先解析你的简历..."}

data: {"type":"tool_call","tool":"parse_resume","status":"running"}

data: {"type":"tool_result","tool":"parse_resume","status":"done"}

data: {"type":"text","content":"你的简历结构比较完整..."}

data: [DONE]
```

**变更点：**
1. 内部不再有子 Agent 的 `AGENT_END` 事件，所有事件均来自主 Agent
2. ChatController 中 `event.getSource() != null` 的子 Agent 事件过滤逻辑可简化
3. ChatRequest.java 需新增 `mode` 字段（当前不存在）

## 8. 风险与注意事项

| 风险 | 应对 |
|------|------|
| 删除 Entity 可能影响其他模块的级联引用 | 逐表检查外键依赖后再 DROP |
| 移除子 Agent 后主 Agent 推理能力下降 | 通过 system prompt 工程补偿，提供更详细的工具使用指南 |
| 前端页面可能依赖已删除的 API | 同步清理前端 API Service 和路由 |
| 现有用户数据（薪资记录、学习计划）丢失 | 提前备份，必要时提供数据导出 |
