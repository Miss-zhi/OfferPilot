# 薪资谈判助手增强

## 1. 背景与目标

### 1.1 背景

需求规格说明书（§119-129）要求 Agent 辅助薪资谈判决策，包括联网搜索薪资范围、多维度对比 offer、生成谈判话术、分析长期价值。当前代码存在以下严重缺陷：

- **联网搜索缺失**：`SalaryService.searchSalary()` 仅查询 MySQL `op_salary_record` 表，不主动联网
- **多维度对比不完整**：`compare_offers` 仅支持 base/months/bonus/stock/location，缺少福利/通勤成本/技术成长/团队水平/行业前景
- **谈判话术严重硬编码**：`SalaryService.generateNegotiationScript()` 的 `buildOpeningLine()`、`buildTalkingPoints()`、`buildClosingLine()` 全部使用 `switch-case` 返回固定中文文本，是 P0 级工具越权生成内容问题
- **长期价值分析完全缺失**：无技术成长/团队水平/行业前景的任何评估逻辑

### 1.2 目标

1. 重构 `search_salary`：DB 无结果时自动触发 `web_search` 联网搜索最新薪资行情
2. 扩展 `compare_offers`：支持通勤成本、福利、技术成长、团队水平、行业前景等维度
3. **彻底去硬编码化**：将 `generateNegotiationScript` 从硬编码 switch-case 改为返回结构化谈判要素 + LLM 指导
4. 新增 `analyze_offer_value` 工具，从技术成长/团队水平/行业前景评估 offer 长期价值

## 2. 功能清单

## 4. 技术方案

### 4.1 整体架构

```
用户输入 offer 详情 / 目标薪资
    ↓
主Agent → spawn salary_advisor
    ↓
salary_advisor 调用链：
  ├── search_salary(company, position)
  │     ├── 先查 MySQL op_salary_record
  │     └── 无结果 → WebSearchFallbackService.search() 联网兜底
  │
  ├── compare_offers(offers_json)
  │     └── 多维度（薪资 + 福利 + 通勤 + 技术成长 + 团队 + 行业）
  │
  ├── analyze_offer_value(company, position, base_salary)
  │     └── 技术成长评分 + 团队水平评估 + 行业前景分析
  │
  └── generate_negotiation_script(current_offer, target_salary, style, market_data)
        └── 去硬编码：返回结构化谈判要素 + LLM 指导
```

### 4.2 search_salary 集成联网兜底

```java
// SalaryService.java 重构

@Service
@RequiredArgsConstructor
public class SalaryService {

    private final SalaryRecordRepository salaryRepo;
    private final WebSearchFallbackService webSearchService;  // 新增注入

    public SalarySearchResult searchSalary(String company, String position) {
        // 1. 先查 DB
        List<SalaryRecord> dbResults = searchDb(company, position);

        if (!dbResults.isEmpty()) {
            return buildResult(dbResults, "db");
        }

        // 2. DB 无结果 → 联网兜底
        log.info("No DB results for {} {}, falling back to web search", company, position);
        String query = String.format("%s %s 薪资 待遇", company, position);
        List<WebSearchFallbackService.WebSearchItem> webResults = webSearchService.search(query, 5);

        if (webResults.isEmpty()) {
            return new SalarySearchResult(0, List.of());
        }

        // 3. 从网页结果中提取薪资信息
        List<SalarySearchResult.SalaryItem> items = webResults.stream()
                .map(r -> {
                    SalarySearchResult.SalaryItem item = new SalarySearchResult.SalaryItem();
                    item.setCompany(company);
                    item.setPosition(position);
                    item.setBaseRange("互联网公开数据");
                    item.setBonusRange("详见下方摘要");
                    item.setSource("web");
                    item.setRelevanceScore(0.7f);
                    // 附加搜索结果文本供 LLM 解读
                    item.setRawContent(r.getContent());
                    return item;
                })
                .toList();

        log.info("Web search salary: company={}, position={}, results={}",
                company, position, items.size());
        return new SalarySearchResult(items.size(), items);
    }
}
```

```java
@Data
public static class OfferItem {
    private String company;
    private String position;

    private Integer welfareScore;      // 福利评分 0-100
    private Integer techGrowthScore;   // 技术成长评分 0-100
    private String teamLevel;          // 团队水平描述
    private String industryProspect;   // 行业前景描述
}

// SalaryService.compareOffers 扩展维度计算
public OfferComparisonResult compareOffers(OfferCompareRequest request) {
    List<OfferCompareRequest.OfferItem> offers = request.getOffers();
    // ... 基础计算 ...

    // 新维度：综合得分计算
    for (OfferCompareRequest.OfferItem offer : offers) {
        double totalPackage = (offer.getBase() != null ? offer.getBase() : 0)
                * (offer.getMonths() != null ? offer.getMonths() : 12);

        // 综合得分 = 年薪(40%) + 福利(15%) + 技术成长(20%) + 通勤(10%) + 股票(15%)
        double compositeScore = calculateCompositeScore(offer, totalPackage);

        OfferComparisonResult.OfferAnalysis analysis = new OfferComparisonResult.OfferAnalysis();
        analysis.setCompany(offer.getCompany());
        analysis.setTotalPackage(totalPackage);
        analysis.setCompositeScore(Math.round(compositeScore));
        analysis.setPros(buildOfferPros(offer));
        analysis.setCons(buildOfferCons(offer));
        // ...
    }
}

private double calculateCompositeScore(OfferCompareRequest.OfferItem offer, double totalPackage) {
    // normalizeBase 的满分值应随岗位级别动态调整，避免初级岗位所有 offer 得分均偏低。
    // TODO: 根据 position 中的级别关键词（初级/高级/资深/专家）动态选择满分值。
    double salaryScore = normalizeBase(totalPackage, 200);  // 假设 200 万为满分（适用于 P7+ 级别）
    double welfareScore = offer.getWelfareScore() != null ? offer.getWelfareScore() : 50;
    double techScore = offer.getTechGrowthScore() != null ? offer.getTechGrowthScore() : 50;
    double commuteScore = calculateCommuteScore(offer.getCommuteCost());
    double stockScore = offer.getStock() != null && !offer.getStock().isBlank() ? 80 : 40;

    return salaryScore * 0.40 + welfareScore * 0.15 + techScore * 0.20
            + commuteScore * 0.10 + stockScore * 0.15;
}
```

### 4.4 generateNegotiationScript 彻底去硬编码化

```java
/**
 * 重构后：返回结构化谈判要素 + LLM 指导。
 * 彻底移除所有 switch-case 固定话术文本。
 */
public NegotiationScriptResult generateNegotiationScript(NegotiationScriptRequest request) {
    String style = request.getNegotiationStyle() != null ? request.getNegotiationStyle() : "moderate";
    double targetSalary = request.getTargetSalary() != null ? request.getTargetSalary() : 0;

    // === 仅返回结构化谈判要素，不包含任何固定话术 ===
    List<NegotiationScriptResult.NegotiationElement> elements = List.of(
        buildElement("锚定效应", "先提出较高但合理的期望薪资，作为谈判起点", 1),
        buildElement("BATNA", "确定自己的最佳替代方案，明确谈判底线", 2),
        buildElement("价值陈述", "列举自身技能、经验、证书等核心价值", 3),
        buildElement("市场对标", "引用同行业同岗位薪资数据作为支撑", 4),
        buildElement("让步梯度", "设计递减的让步幅度（如：20% → 10% → 5%）", 5),
        buildElement("非金钱要素", "探讨签字费、搬迁补贴、弹性工作等替代补偿", 6)
    );

    // 组装 LLM 指导文本（不包含成品话术）
    String guidance = buildScriptGuidance(style, targetSalary, elements,
            request.getCurrentOffer(), request.getMarketData());

    return new NegotiationScriptResult(guidance, elements, style);
}

/** 构建谈判要素 — 仅提供结构化信息 */
private NegotiationScriptResult.NegotiationElement buildElement(
        String name, String description, int sequence) {
    NegotiationScriptResult.NegotiationElement e = new NegotiationScriptResult.NegotiationElement();
    e.setName(name);
    e.setDescription(description);
    e.setSequence(sequence);
    return e;
}

/** 构建 LLM 指导 — 告知 LLM 需要组装什么，但不提供成品文本 */
private String buildScriptGuidance(String style, double targetSalary,
        List<NegotiationScriptResult.NegotiationElement> elements,
        String currentOffer, String marketData) {

    String styleDesc = switch (style) {
        case "assertive" -> "强硬风格：直接表达期望，强调自身价值和市场竞争力";
        case "moderate" -> "温和风格：平衡双方利益，寻求共赢方案";
        default -> "保守风格：以了解信息为主，表达合作意愿";
    };

    return String.format("""
            请根据以下要素生成一段专业的薪资谈判话术（%s）：
            期望薪资：%.0f 万/年
            当前 offer：%s
            市场数据：%s

            请按以下顺序组织话术：
            %s

            要求：
            - 语言专业礼貌，符合%s特点
            - 从第1步锚定效应开始，逐步推进到第6步
            - 每步1-3句话，整体不超过300字
            - 体现专业谈判技巧但不要像在读谈判课教材
            """,
            styleDesc, targetSalary, currentOffer, marketData,
            formatElements(elements), styleDesc);
}

/** 将谈判要素格式化为步骤列表 */
private String formatElements(List<NegotiationScriptResult.NegotiationElement> elements) {
    StringBuilder sb = new StringBuilder();
    for (var e : elements) {
        sb.append(String.format("%d. 【%s】%s\n", e.getSequence(), e.getName(), e.getDescription()));
    }
    return sb.toString();
}
```

### 4.5 OfferValueTool（长期价值分析）

```java
@Slf4j
@Component
@RequiredArgsConstructor
public class OfferValueTool {

    @Tool(name = "analyze_offer_value",
          description = "从技术成长、团队水平、行业前景三个维度评估 offer 的长期价值")
    public OfferValueResult analyze(
            @ToolParam(name = "company", description = "公司名称") String company,
            @ToolParam(name = "position", description = "岗位名称") String position,
            @ToolParam(name = "base_salary", description = "月薪（K）") Double baseSalary) {

        log.info("analyze_offer_value: company={}, position={}", company, position);

        OfferValueResult result = new OfferValueResult();
        result.setCompany(company);
        result.setPosition(position);

        // 工具只提供公司/岗位信息和评估维度，不做硬编码判断。
        // 公司评级、行业判断交给 LLM，避免固定公司名单导致的覆盖不全和偏见。
        String guidance = String.format("""
                请从以下三个维度评估该 offer 的长期价值，给出 0-100 评分和简要分析：

                【公司信息】%s
                【目标岗位】%s
                【月薪参考】%.0fK

                评估维度：
                1. 技术成长空间（0-100）：考虑公司技术栈先进性、是否有 mentor 制度、
                   个人成长曲线陡峭程度
                2. 团队水平（0-100）：考虑团队成员背景、技术氛围、协作文化
                3. 行业前景（0-100）：考虑所处行业增长趋势、公司发展阶段、市场竞争力

                请结合你对该公司/行业的了解给出客观评分和简短评语。
                如果信息不足，请基于岗位和月薪水平做出合理推断并标注不确定性。""",
                company, position, baseSalary != null ? baseSalary : 0);

        result.setGuidance(guidance);
        return result;
    }
}
```

### 4.6 AgentFactory 变更

```java
private SubagentDeclaration buildSalaryAgent() {
    return SubagentDeclaration.builder()
        .name("salary_advisor")
        .description("薪资谈判顾问...")
        .inlineAgentsBody("""
            你是资深HR和职业规划顾问。

            工作流程：
            1. 调用 search_salary 检索薪资数据（自动含联网兜底）
            2. 调用 compare_offers 进行多维度 offer 对比
            3. 调用 analyze_offer_value 评估各 offer 的长期价值
            4. 调用 generate_negotiation_script 获取谈判要素和指导
            5. 基于以上结构化数据，整合输出：
               - 薪资行情概览
               - Offer 多维度对比表
               - 长期价值分析
               - 个性化谈判策略和话术
            6. 如果本地数据不足，调用 web_search 补充行业数据和公司评价

            薪资信息敏感，回复要专业、客观、有数据支撑。""")
        .tools(List.of("search_salary", "compare_offers",
            "generate_negotiation_script", "analyze_offer_value", "web_search"))
        .build();
}
```
