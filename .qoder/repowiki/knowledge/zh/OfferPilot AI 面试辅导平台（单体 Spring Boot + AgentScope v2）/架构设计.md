单体 Spring Boot 应用，入口 `OfferPilotApplication` 启用 JPA Auditing/JPA Repositories/EntityScan/Scheduling。包结构严格遵循四层边界：
- controller/：7 个 REST Controller（Auth/Chat/KnowledgeBase/ModelConfig/Report/SearchStats/UserModel），仅做参数校验与响应封装，不直接访问 Repository。
- service/：15+ 业务 Service，编排领域逻辑；其中 ingestion/ 子包封装文档分片与入库流程，ApiKeyEncryption 负责敏感字段加解密。
- agent/：AgentScope v2 集成层，`AgentFactory` 通过 Caffeine 有界缓存池管理 HarnessAgent 实例，按用户维度构建 Toolkit（10 个本地 @Tool + MCP web-search）、Permission 白名单、Middleware（TokenMonitor/CostControl）；模型解析优先级为「用户私有 > 用户默认 > 全局默认 > application.yml」。
- entity/repository/：18 个 JPA Entity + 15 个 Repository，对应 MySQL 持久化。

外部依赖方向：controller → service → (agent/service) → repository；agent 层反向依赖 service（UserMemoryService/UserModelService/ModelConfigService）注入业务数据。配置采用 application.yml + application-dev.yml + application-prod.yml 多环境策略，敏感信息通过 ${ENV} 占位符注入。RAG 知识库走 Milvus 向量库（动态 Collection + 通用 Schema），Redis 用于会话状态与搜索缓存，H2 仅 dev 环境使用。
