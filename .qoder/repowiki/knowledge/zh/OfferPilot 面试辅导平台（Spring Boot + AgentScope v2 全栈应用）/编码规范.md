- 异常体系以 `BusinessException` 为基类（errorCode + message），派生 DuplicateUserException(409)、InvalidCredentialsException(401)、AccountDisabledException(403)、RateLimitException(429) 等子类，统一由 `GlobalExceptionHandler` 的 `@RestControllerAdvice` 捕获并返回 `ApiResponse`。
- Agent 工具类集中在 `agent/tool/` 下，每个工具一个独立类并通过 `AgentFactory.buildToolkit()` 按功能分组注册，禁止在 Controller 中直接调用工具。
- 配置项集中定义在 `AgentScopeProperties` / `MilvusProperties` 等 POJO 中，通过 `@ConfigurationProperties(prefix = "agentscope.*")` 绑定，避免在代码中散落字符串 key。
- 测试数据约定 create_by='test' 或 id>=99000，单元测试用 JUnit 5 + Mockito，集成测试用 Testcontainers 启动真实 MySQL/Milvus/Redis 容器，API 端点测试用 REST Assured + MockMvc，安全场景用 `@WithMockUser`。
- DTO 按领域分包（auth/chat/kb/model/salary/tool），每个请求/响应对象独立文件，Controller 与 Service 之间不直接透传 Entity。
