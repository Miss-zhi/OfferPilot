## 错误处理架构概述

OfferPilot 项目采用基于 Spring Boot 的集中式异常处理架构，通过自定义异常类型、全局异常处理器和统一的响应格式实现结构化的错误管理。

## 核心组件

### 1. 基础异常体系
- **BusinessException**: 所有业务异常的基类，继承自 RuntimeException，包含 errorCode 和 message 两个核心字段
- **具体业务异常**: 继承 BusinessException 的特定场景异常类，每个类封装固定的 HTTP 状态码和业务语义

### 2. 全局异常处理器
**GlobalExceptionHandler** 使用 `@RestControllerAdvice` 注解实现集中式异常捕获：
- `handleBusinessException`: 处理所有业务异常，返回对应的 HTTP 状态码
- `handleValidationException`: 处理参数校验失败，聚合多个字段错误信息
- `handleIllegalArgument`: 处理非法参数异常，返回 500 内部错误
- `handleException`: 兜底处理所有未捕获异常

### 3. 统一响应格式
**ApiResponse<T>** 提供标准化的响应结构：
- code: 业务状态码（200成功，其他为错误码）
- message: 人类可读的错误描述
- data: 业务数据载荷

## 具体异常类型设计

| 异常类 | HTTP状态码 | 使用场景 |
|--------|------------|----------|
| ResourceNotFoundException | 404 | 资源不存在 |
| DuplicateUserException | 409 | 用户重复注册 |
| InvalidCredentialsException | 401 | 认证失败 |
| AccountDisabledException | 403 | 账号被禁用 |
| FileUploadException | 400 | 文件上传错误 |
| RateLimitException | 429 | 频率限制触发 |

## 错误处理模式

### 服务层异常抛出
```java
// 业务逻辑验证失败
if (userRepository.existsByUsername(req.getUsername())) {
    throw new DuplicateUserException();
}

// 权限检查失败
throw new BusinessException(403, "无权删除此知识库");

// 外部依赖错误
throw new RuntimeException("Embedding API 返回错误: " + response.code());
```

### Controller 层集成
- 在控制器方法中直接抛出业务异常
- 结合 Spring Validation 进行参数校验
- SSE 流式响应中的异步错误处理

## 设计决策与约定

1. **异常层次清晰**: 所有业务异常都继承自 BusinessException，便于统一处理
2. **HTTP状态码映射**: 每个异常类对应明确的 HTTP 状态码，符合 RESTful 规范
3. **错误信息国际化准备**: 异常消息使用中文，便于前端直接展示
4. **日志记录策略**: 不同级别的异常使用不同的日志级别（warn/error）
5. **防御性编程**: 对第三方调用使用 try-catch 包裹并转换为运行时异常

## 开发规范建议

1. 优先使用具体的业务异常类而非通用异常
2. 在 Service 层抛出异常，Controller 层只负责接收和转换
3. 避免在异常消息中泄露敏感信息
4. 对于可恢复的错误使用合适的 HTTP 状态码
5. 记录详细的错误上下文信息用于问题排查
