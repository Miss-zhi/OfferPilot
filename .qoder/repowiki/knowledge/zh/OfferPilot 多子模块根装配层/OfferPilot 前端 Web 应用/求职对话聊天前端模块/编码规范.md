- 所有对外暴露的状态与接口均通过独立 `.ts` 文件中的 `interface` 显式声明（如 `ChatMessage`、`ChatState`、`StreamCallbacks`），避免在组件内定义类型。
- Zustand store 使用 `create<T>((set) => ({ ... }))` 工厂模式，每个 setter 都返回新的不可变对象（展开数组或对象），不使用 `state.messages.push` 这类可变操作。
- SSE 流式处理统一通过 `onmessage` 中按 `event.event` 值分派到 `onDelta/onDone/onError` 回调，错误路径一律调用 `callbacks.onError` 并 `throw err` 停止重连。
- 功能入口通过 `as const` 常量数组（`CHAT_FUNCTIONS`）集中声明 key/label/icon，UI 侧通过 `iconMap` 映射表将字符串图标名解析为 React 节点，实现配置与渲染解耦。
