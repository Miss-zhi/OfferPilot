- 所有对外 HTTP 调用统一走 `@/service/kbService`，并通过 `App.useApp().message` 在 try/catch 中给出 success/error/warning 提示，失败分支一律 catch 后返回而不抛错。
- 表格列渲染使用 `ColumnsType<T>` 泛型 + `render` 回调，枚举值（visibility/status/chunkStrategy/doc status）通过本地映射表或 switch 转为 `<Tag color=...>` 展示。
- 需要确认危险操作（删除、批量删除）时使用 Antd 的 `<Popconfirm>` 包裹确认后再执行 API 调用。
- 跨页面共享当前知识库上下文采用 Zustand store `useKbStore`，进入详情页前通过 `setCurrentKb(kbId, name)` 写入，详情页仅读取 `currentKbName` 做面包屑展示。
- 异步副作用统一用 `useCallback` 包裹并放入 `useEffect` 依赖数组，组件卸载时在 cleanup 中清理定时器（如 `pollingRefs.current.forEach(clearInterval)`）。
