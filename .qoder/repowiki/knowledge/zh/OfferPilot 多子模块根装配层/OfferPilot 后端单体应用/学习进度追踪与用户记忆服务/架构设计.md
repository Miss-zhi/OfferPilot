本模块由三层组成：
- REST 层 `ProgressController` 暴露 `/api/v1/offerpilot/progress` GET 端点，直接依赖三个 JPA Repository（`InterviewSessionRepository`、`KnowledgeMasteryRepository`、`StudyPlanRepository`），在 Controller 内完成多表数据拼装（分数趋势、知识点掌握度、学习计划进度），未下沉到 Service 层。
- 业务层 `UserMemoryService` 封装对 `UserMemoryRepository` 的 CRUD，提供 `loadUserMemory`（按相关性排序分组拼接为 Markdown 片段）、`saveMemory`（upsert by key）、`removeMemory` 三个方法，读写均加 `@Transactional`；读取时同步更新访问计数与最近访问时间，用于热度衰减。
- DTO 层 `dto.progress.ProgressResponse` 使用 Lombok `@Data`，并通过嵌套静态类 `MasteryInfo`、`StudyPlanInfo` 表达响应结构。
依赖方向：Controller → Repository + DTO，Service → Repository，无跨模块外部依赖，属于纯内存+JPA 的数据聚合与记忆存取子域。
