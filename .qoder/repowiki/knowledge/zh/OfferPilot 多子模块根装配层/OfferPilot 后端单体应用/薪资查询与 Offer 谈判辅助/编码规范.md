- Controller 使用 @RequiredArgsConstructor + final 字段注入 Service，并通过 @Slf4j 记录关键入参后再调用 Service。
- 所有对外请求 DTO 使用 Lombok @Data + Jakarta Validation 注解（@NotBlank/@NotEmpty）配合 Controller 层的 @Valid 进行入参校验。
- Service 将复杂业务拆分为多个 private 构建方法（如 buildOfferPros/buildOfferCons/buildOpeningLine 等），每个方法只负责单一职责的片段生成。
- Controller 统一通过 ApiResponse.success(...) 包装返回值，不在控制器内自行构造 HTTP 状态码。
