本模块遵循项目统一的 controller → service → repository 分层：
- `SalaryController` 暴露 `/api/v1/offerpilot/salary` 下的三个端点（GET /search、POST /compare、POST /negotiation-script），仅做参数校验（@Valid）与日志记录，调用 Service 后统一通过 `ApiResponse.success()` 返回。
- `SalaryService` 是纯业务层，依赖 `SalaryRecordRepository` 读取 `SalaryRecord` 实体；内部以私有方法拆分子逻辑（`formatSalaryRange`、`buildOfferPros/Cons`、`buildOpeningLine/TalkingPoints/CounterArguments/ClosingLine`），不直接访问外部 Agent 或 RAG 库，属于轻量计算型服务。
- `dto/salary/` 包存放请求 DTO（`NegotiationScriptRequest`、`OfferCompareRequest`、`SalarySearchRequest`），其中 `OfferCompareRequest.OfferItem` 作为嵌套静态类承载单个 offer 字段；响应 DTO 来自 `dto.tool.*`（`SalarySearchResult`、`OfferComparisonResult`、`NegotiationScriptResult`），由 Service 组装后返回。
- 依赖方向单向：Controller → Service → Repository，DTO 仅用于入参/出参，无循环引用。
