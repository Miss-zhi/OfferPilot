项目采用标准四层分层：controller/（10 个 REST 控制器，仅做参数校验与响应封装，不直接访问 Repository）→ service/（15+ 业务 Service，编排 JPA、Milvus、Redis、外部 LLM 调用）→ agent/（AgentScope v2 HarnessAgent 工厂 + 13 个 @Tool + 2 个 Middleware）→ entity/repository/（18 个 JPA Entity + 15 个 Repository）。

- 入口：`OfferPilotApplication` 启用 `@EnableJpaAuditing`、`@EnableScheduling`，扫描 `entity` 与 `repository`。
- Agent 层：`AgentFactory` 通过 Caffeine 有界缓存池管理 per-user 的 `HarnessAgent` 实例，按「用户私有模型 > 用户默认 > 全局默认 > application.yml」优先级解析 Model，并通过 `PermissionContextState` 白名单放行工具调用；7 个子 Agent 以 `SubagentDeclaration.inlineAgentsBody` 内联定义，主 Agent 仅作为调度中心禁止直调业务工具。
- RAG 知识库：`KnowledgeBaseService` 实现「多路召回→融合排序→MCP 兜底」搜索管线（PUBLIC KB 向量合并检索 → InterviewQuestion DB LIKE → WebSearchFallback MCP），并集成 `PersonalizedRankService` 按用户薄弱点加权排序；`MilvusCollectionManager` 负责 Collection 生命周期（创建含 IVF_FLAT+COSINE 索引 + Load 到内存）。
- 安全：`SecurityConfig` + `JwtAuthenticationFilter` + `JwtTokenProvider`（jjwt 0.12.5）+ `CustomUserDetailsService`，异常由 `GlobalExceptionHandler` 统一捕获返回 `ApiResponse`。
- 配置：`application.yml`（公共）+ `application-dev.yml`（H2 + DEBUG）+ `application-prod.yml`（MySQL + INFO），敏感信息通过 `${ENV}` 注入；测试使用 `application-test.yml` + Testcontainers 启动真实 MySQL/Milvus/Redis。
- 前端：`web/` 为独立 Vite + React + TypeScript SPA，通过 `infra/http.ts` 调用后端 REST API，构建产物由后端静态资源托管。
