- Controller 仅做入参校验与结果转换，所有数据访问与业务编排下沉到 Service 层，禁止 Controller 直接注入 Repository。
- 业务异常继承 `BusinessException`（errorCode + message），由 `GlobalExceptionHandler` 统一映射为对应 HTTP 状态码与 `ApiResponse` 响应体。
- Agent 工具类统一放在 `agent/tool/` 下并以 `*Tool.java` 命名，通过 `AgentFactory.buildToolkit()` 按功能分组（knowledge_retrieval/resume_analysis/interview/utility）注册到 Toolkit。
- 搜索接口遵循「向量检索 → DB 回退 → MCP 联网兜底」三段式管线，并在最后统计各来源命中数供 `SearchAnalyticsService` 异步记录。
- 配置文件采用多环境策略：`application.yml` 存放公共配置，dev/prod/test 覆盖差异化项，所有密钥通过 `${ENV_VAR:default}` 占位符注入，不在文件中硬编码。
- 测试分为单元测试（JUnit5 + Mockito）、集成测试（`AbstractServiceIT`/`AbstractControllerIT` + Testcontainers 启动真实 MySQL/Milvus/Redis）、API 端点测试（REST Assured + MockMvc），测试数据约定 create_by='test' 或 id>=99000。
