模块按职责切分为四个子包，彼此单向依赖：
- `common`：纯数据载体，无 Spring 注解，被 controller/service/exception 共同引用。`ApiResponse<T>` 通过静态工厂 `success()/error()` 构造统一 REST 响应；`BaseEntity` 以 `@MappedSuperclass` + `AuditingEntityListener` 注入 id/createdAt/updatedAt/createBy/updateBy 审计字段；`PageRequest` 封装分页参数并计算 offset。
- `config`：全部以 `@Configuration` + `@Bean` 暴露第三方客户端，不持有业务状态。`SecurityConfig` 禁用 CSRF、设置 STATELESS 会话、注册 `JwtAuthenticationFilter` 到 `UsernamePasswordAuthenticationFilter` 之前，并通过 `@EnableMethodSecurity` 启用方法级授权；`WebConfig` 仅对 `/api/**` 开放 CORS；`MilvusConfig`/`RedisConfig`/`AsyncConfig` 分别构建 MilvusClientV2、RedisTemplate 和线程池。
- `security`：基于 `OncePerRequestFilter` 的无状态 JWT 过滤器链。`JwtTokenProvider` 使用 jjwt 0.12+ 的 `Jwts.builder()`/`parser().verifyWith(key)` 进行 HMAC-SHA 签名与校验，密钥与过期时间通过 `@Value("${app.jwt.secret}")` 注入；`CustomUserDetailsService` 从数据库加载 `UserDetails`；`JwtAuthenticationFilter` 解析 Authorization Bearer 头后写入 `SecurityContextHolder`。
- `exception`：`BusinessException` 作为基类携带 `errorCode`，派生出 DuplicateUser(409)/InvalidCredentials(401)/AccountDisabled(403)/RateLimit(429) 等业务异常；`GlobalExceptionHandler` 通过 `@RestControllerAdvice` 统一捕获 BusinessException、`MethodArgumentNotValidException`、`IllegalArgumentException` 及兜底 Exception，均返回 `ApiResponse`。
依赖方向：`config` → `security` → `common`；`exception` → `common`；其余层均可引用 `common`，但本模块不反向依赖业务层。
