模块以 AgentFactory 为唯一入口：构造器注入所有 11 个 @Component Tool 实例与 AgentScopeProperties、UserMemoryService、ModelConfigService、ApiKeyEncryption 等 service 层依赖；buildToolkit() 通过 Toolkit.createToolGroup 划分四个语义分组（knowledge_retrieval / resume_analysis / interview / utility），再以 toolkit.registration().tool(...).group(组名).apply() 批量绑定并统一 registerMetaTool() 暴露 plan_enter 等元能力；buildAgent() 按 userId 优先级解析 Model（私有 > 默认 > 全局 > application.yml 兜底），组装 TokenMonitorMiddleware + CostControlMiddleware 中间件链，启用 enablePlanMode() + enableTaskList() 支持断点续聊与 todo 管理；getOrCreateAgent(userId) 经 Caffeine 有界缓存（最大 500、30 分钟未访问淘汰）复用或重建。

middleware/ 下两个中间件实现 MiddlewareBase 接口：TokenMonitorMiddleware 在 onReasoning/onModelCall/onActing 钩子采集推理轮次、prompt/completion token 与工具调用耗时；CostControlMiddleware 在 onAgent 重置计数器、onModelCall 拦截并拒绝超会话次数（50 次）或超 Token 阈值（20 万）的调用，抛出内部 CostLimitExceededException 由上层捕获。

tool/ 下每个工具类仅依赖 service/ 层（如 ResumeParseTool → ResumeService），不直接访问 entity/repository，也不被 tool 反向引用，形成「controller → service → agent/tool」单向依赖边界。
