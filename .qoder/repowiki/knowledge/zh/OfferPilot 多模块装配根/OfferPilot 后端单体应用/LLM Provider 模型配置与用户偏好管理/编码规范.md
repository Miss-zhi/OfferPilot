- Service 层所有写操作使用 @Transactional 注解，读操作不加事务注解，保持读写分离的事务粒度。
- 业务校验失败统一抛出 BusinessException(HttpStatus.xxx.value(), message)，由全局异常处理器转为 ApiResponse 错误响应。
- DTO 构建统一使用 Lombok @Builder，Entity 到 Response 的转换集中在 Service 私有的 toResponse/buildPrivateModelResponse 方法中完成。
- 对外暴露的 API Key 一律通过 ApiKeyEncryption.mask() 脱敏后再放入响应体，禁止明文回传。
- 新增/更新配置时先通过 ProviderPreset.fromProviderKey() 解析预设并填充 baseUrl/apiFormat/authHeaderType 等字段，避免硬编码。
