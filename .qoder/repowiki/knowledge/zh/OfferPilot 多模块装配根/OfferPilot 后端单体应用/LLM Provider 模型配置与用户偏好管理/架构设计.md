本模块遵循 controller → service → repository 分层：
- Controller 层：`ModelConfigController`（/api/v1/admin/models，@PreAuthorize("hasRole('ADMIN')")）负责管理员侧模型池管理；`UserModelController`（/api/v1/user/models）通过 @AuthenticationPrincipal 获取当前用户，暴露用户可用模型列表、设置默认模型、创建私有模型三个端点。
- Service 层：`ModelConfigService` 编排管理员侧业务（创建/更新/删除配置、刷新模型名称、设置全局默认），`UserModelService` 处理用户侧逻辑（查询可用模型、设置用户默认、创建私有模型并自动设为默认）。两者均依赖 `ApiKeyEncryption` 做 AES 加解密、`ModelListFetcher` 调用远端 Provider API 拉取模型列表。
- 外部适配：`ModelListFetcher` 基于 JDK HttpClient + Jackson JsonNode 解析 OpenAI / Anthropic / Gemini 三种响应格式；`ProviderPreset` 枚举集中维护 8 家 Provider 的默认 Base URL、模型列表 URL、API 格式与认证头类型，作为配置预设来源。
- 数据层：`ModelConfig`（op_model_config）存储 Provider 接入参数与启用/默认/私有标记，`ModelName`（op_model_name）缓存从 Provider 拉取的模型名列表，二者均继承 BaseEntity，Repository 提供按 isEnabled/isGlobalDefault/provider/modelConfigId 等条件的查询方法。
- 安全边界：API Key 在持久化前经 `ApiKeyEncryption.encrypt()` 写入数据库，返回 DTO 时经 `mask()` 脱敏；敏感密钥通过环境变量 app.encryption.secret-key 注入。
