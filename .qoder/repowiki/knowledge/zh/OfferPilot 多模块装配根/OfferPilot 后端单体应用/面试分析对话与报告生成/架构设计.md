本模块由三层组成：
- REST 入口 `ChatController`（`/api/v1/offerpilot/chat`）：暴露 `/chat`（同步）和 `/stream`（SSE 流式）两个端点，统一通过 `@AuthenticationPrincipal` 获取 userId、调用 `RateLimitService.tryAcquireDialogue` 做频率控制，再通过 `AgentFactory.getOrCreateAgent(userId)` 获取对应租户的 `HarnessAgent`，并以 `RuntimeContext(userId, sessionId)` 驱动 Agent。
- Agent 交互层：同步路径直接 `agent.call(...).block()` 取文本；流式路径订阅 `agent.streamEvents(...)` 的 `Flux<AgentEvent>`，仅转发 `TEXT_BLOCK_DELTA`（事件名 `delta`）和 `AGENT_END`（事件名 `done`），错误走 `error` 事件，5 分钟超时 `SseEmitter(300_000L)`。
- 分析持久化层 `InterviewAnalysisService`：负责将 AI 给出的评分写入 `InterviewQuestion`、按知识点更新 `KnowledgeMastery`，以及从 Session+Questions 聚合计算平均分、维度与改进建议后落库为 `AnalysisReport`，所有写操作标注 `@Transactional`。
依赖方向严格遵循项目约定：Controller → Service → Repository，不跨层访问数据源；DTO 仅位于 `dto.chat` 包内，作为 HTTP 契约边界。
