- UI 组件内异步操作统一采用 try/catch/finally 包裹，在 finally 中重置 loading 状态，错误通过 antd message.error 提示。
- 表单字段校验规则以数组形式内联声明在 Form.Item 的 rules 属性中，包含 required、min/max 等基础约束。
- 服务层方法返回 Promise，并通过 .then(res => res.data) 剥离 axios 响应包装，对上层只暴露业务数据接口。
- Zustand store 通过 persist 中间件配合 STORAGE_KEYS 常量名进行本地持久化，state 初始值与 set/logout 中对 localStorage 的操作保持同步。
