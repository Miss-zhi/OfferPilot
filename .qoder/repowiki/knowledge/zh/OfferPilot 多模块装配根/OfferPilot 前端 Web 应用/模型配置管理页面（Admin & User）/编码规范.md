- service 层以单一对象导出（`export const modelConfigService = { ... }`），方法名采用动词+名词形式（如 `listConfigs`、`createConfig`），并通过 `http.get/post/put/delete<T>(...).then(res => res.data)` 链式返回解包后的数据。
- 前端类型定义集中在 service 文件中，使用 `interface` 显式声明请求体与响应体（`CreateModelConfigRequest`、`UpdateModelConfigRequest`、`SetGlobalDefaultRequest` 等），避免在组件中散落 any。
- 页面内每个异步操作遵循 try/catch + `message.success/error` 反馈 + 成功后 `loadData()` 刷新列表的统一错误处理模式。
- 表单使用独立的 `Form.useForm()` 实例（创建/编辑/全局默认各一个），提交前调用 `validateFields()` 校验，关闭 Modal 时统一 `resetFields()` 清理状态。
