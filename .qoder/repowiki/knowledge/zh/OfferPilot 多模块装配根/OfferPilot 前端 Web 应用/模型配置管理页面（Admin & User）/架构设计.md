本模块由一个纯展示型 React 组件与一个服务层文件组成，职责清晰分离：
- `ModelConfigPage.tsx` 是 Ant Design 驱动的无状态页面组件，通过 `useState`/`useEffect`/`useCallback` 管理本地 UI 状态（列表、预设、三个 Modal 表单），所有业务交互最终委托给 `modelConfigService`。
- `modelConfigService.ts` 作为单一导出对象，按「管理员 API」和「用户 API」两个命名空间组织方法，统一通过 `@/infra/http` 发起请求并以 `API.ADMIN_*` / `API.USER_*` 常量拼接路由，同时集中声明前后端共享的 TypeScript 接口（`ModelConfigItem`、`CreateModelConfigRequest`、`ProviderPresetItem` 等）。
依赖方向单向：页面 → service → infra/http + constants；不直接访问后端，也不持有跨组件状态。
