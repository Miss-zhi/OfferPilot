本模块是 OfferPilot 的浏览器端入口，采用「基础设施层 → 领域服务层 → UI 组件/页面」的分层组织：
- `app/` 提供 `main.tsx` 启动点与全局路由守卫，`infra/http.ts` 封装 Axios 拦截器（统一注入 JWT、解包后端 ApiResponse），`infra/constants.ts` 集中定义 API 路径常量；
- `service/` 下每个文件对应一个后端 Controller 的服务封装（auth/chat/kb/resume/salary/report/progress），被各页面 store 调用；
- `store/` 使用 Zustand 维护跨组件共享状态（auth-store、chat-store、kb-store），由页面组件订阅；
- `ui/pages/` 按业务域划分目录（login、chat、admin/*、resume、salary、reports、progress），每个页面组合若干 `ui/components` 中的可复用原子组件（ChatBubble、FileUploader、MarkdownRenderer、SseStreamRenderer 等）。

跨模块协作契约：所有子模块均通过 `@/infra/http` 发起请求，通过 `@/store/*` 共享状态，通过 `react-router-dom` 注册路由；Vite 配置在 `vite.config.ts` 中统一设置 `@` 别名与 `/api` 代理到 `localhost:8080`，使前后端联调无需额外网关。
