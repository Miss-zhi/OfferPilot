模块按 controller → service → ingestion 子包分层：
- `KnowledgeBaseController` 暴露 `/api/v1/kb` REST 接口，仅做参数校验与日志，所有业务编排委托给 `KnowledgeBaseService`。
- `KnowledgeBaseService` 是核心编排层：负责知识库权限校验（PUBLIC 公共库 + PRIVATE 私有库）、文档生命周期管理、Milvus Collection 级联增删、以及面向 Agent 的四种搜索方法（`searchQuestions`/`searchAnswers`/`searchCompanyInterviews`/`searchResources`），这些方法先查 PUBLIC 知识库的 Milvus，无结果则回退到 `InterviewQuestionRepository` 表内模糊匹配。
- `service.ingestion` 子包实现异步入库管道：`DocumentIngestionService` 通过 `@Async("ingestionExecutor")` 驱动 PARSING→CHUNKING→EMBEDDING→INDEXING→ACTIVE 五阶段状态机；`DocumentParser` 基于 Apache PDFBox + POI 解析 markdown/txt/pdf/docx；`DocumentChunker` 支持 BY_QUESTION/BY_HEADING/BY_SIZE/AUTO 四种策略，大小与重叠量由 `agentscope.knowledge.chunk-*` 配置注入。
- `KbConverter` 作为独立转换器组件，隔离 Entity ↔ DTO 映射。
- 依赖方向单向：Controller → Service → (Repository / MilvusClientV2 / EmbeddingService / FileService)；ingestion 子包不反向依赖 Controller。
