模块围绕 AgentFactory 单例展开：通过 Spring 构造器注入所有 11 个 @Component 标注的 Tool 实例，在 buildToolkit() 中用 Toolkit.createToolGroup 划分四个语义分组（knowledge_retrieval / resume_analysis / interview / utility），并统一调用 toolkit.registerMetaTool() 暴露 plan_enter 等元能力；buildAgent() 以 userId 为键经 Caffeine 有界缓存（最大 500、30 分钟未访问淘汰）返回或创建 HarnessAgent，启用 enablePlanMode() + enableTaskList() 支持断点续聊与 todo 管理。

中间件采用 AgentScope MiddlewareBase 洋葱模式：TokenMonitorMiddleware 在 onReasoning/onModelCall/onActing 三个钩子采集推理轮次、prompt/completion token 与工具调用耗时；CostControlMiddleware 在 onAgent 重置计数器、onModelCall 拦截并拒绝超会话次数（50 次）或超 Token 阈值（20 万）的调用，抛出内部 CostLimitExceededException 由上层捕获。

依赖方向单向向外：本模块仅依赖 service/ 层（UserMemoryService、ResumeService 等）与 config.AgentScopeProperties，不直接访问 entity/repository，也不被 tool 反向引用，形成「controller → service → agent/tool」的清晰分层。
