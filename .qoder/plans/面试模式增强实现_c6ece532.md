## Task 3: 新增 InterviewModeService

**文件**: `src/main/java/com/tutorial/offerpilot/service/InterviewModeService.java`（新建）

实现三个核心方法：

1. **`getPhaseSequence(String mode)`** — 返回各模式下的 12 阶段序列：
   - `TECH_DEEP`: 自我介绍→基础知识→项目经验×3→系统设计→项目经验→底层原理→架构设计→性能优化→项目经验→职业规划
   - `BEHAVIOR`: 自我介绍→项目经验→行为面试×2→项目经验→情景分析→行为面试→压力测试→项目经验→行为面试→情景分析→职业规划
   - `SYSTEM_DESIGN`: 自我介绍→基础知识→系统设计×2→基础知识→系统设计→架构设计→系统设计→性能优化→系统设计→架构设计→职业规划
   - `PRESSURE`: 自我介绍→专业技能×2→项目经验→压力测试→情景分析→压力测试→项目经验→压力测试→专业知识→压力测试→行为面试

2. **`determineDifficulty(String mode, Double avgScore, int questionNumber)`** — 模式感知难度：
   - `PRESSURE`: 起步 medium，均分 >= 60 或 3 题后直接 hard
   - `SYSTEM_DESIGN`: 渐进式（前2 easy, 3-5 medium, 6+ hard）
   - 默认: 沿用现有逻辑（均分 >= 75 hard, >= 50 medium, 或按题号递进）

3. **`extractProjectKeywords(String resumeText)`** — 从简历文本提取项目相关行（不做技术关键词识别，交给 LLM）

## Task 4: 重构 MockInterviewTool.generateNextQuestion()

**文件**: `src/main/java/com/tutorial/offerpilot/agent/tool/MockInterviewTool.java`

1. **方法签名变更** — 新增两个可选参数：
   ```java
   @ToolParam(name = "mode", required = false) String mode,
   @ToolParam(name = "resume_text", required = false) String resumeText
   ```

2. **模式解析逻辑** — `resolveMode(mode, sessionId)`:
   - 优先使用传入的 `mode` 参数
   - 否则从 DB `session.interviewMode` 读取
   - 最终默认 `TECH_DEEP`

3. **阶段判定重构** — `determineCategory` 改为 `determineCategoryByMode`:
   - 调用 `InterviewModeService.getPhaseSequence(mode)` 获取序列
   - 按题号轮转取值
   - 保留 context 关键词的短路逻辑（如 context 明确说要"自我介绍"）

4. **难度判定重构** — `determineDifficulty` 改为 `determineDifficultyByMode`:
   - 调用 `InterviewModeService.determineDifficulty(mode, avgScore, questionNumber)`

5. **出题指导增强** — `buildGuidance` 改为 `buildGuidanceByMode`:
   - `TECH_DEEP` + 有简历: 附上项目关键词，要求 LLM 从中识别技术栈并深挖
   - `TECH_DEEP` + 无简历: 要求深入追问技术原理
   - `BEHAVIOR`: STAR 框架提问
   - `SYSTEM_DESIGN`: 开放型设计题
   - `PRESSURE`: 有挑战性问题，回答后质疑

6. **NewQuestionResult 构造** — 传入 `effectiveMode`

7. **注入 InterviewModeService** — 在构造器中添加
