
# 面试录音六维分析增强

## 概述

将面试分析从 3 维（专业深度/表达能力/内容覆盖度）扩展至 6 维（新增知识盲区/时间分配/自信度），通过 3 个新工具 + AgentFactory 子Agent 工具白名单更新 + InterviewAnalysisService 维度扩展实现。

---

## Task 1: 新增 DTO 文件（3 个）

创建 `dto/tool/` 下的 3 个 Result DTO：

- **KnowledgeGapResult.java** — 知识盲区检测结果
  - 字段：`guidance`(String)、`covered`(List\<String\>)、`missing`(List\<String\>)、`coverageRate`(int)
  - 遵循现有 `AnswerAnalysisResult` 的 DTO 风格（`@Data` + `@NoArgsConstructor` + `@AllArgsConstructor`）

- **TimeAllocationResult.java** — 时间分配分析结果
  - 字段：`guidance`(String)、`items`(List\<TimeItem\>)、`totalSeconds`(int)、`tooShortCount`(int)、`tooLongCount`(int)
  - 内部静态类 `TimeItem`：`question`、`charCount`、`estimatedSeconds`、`assessment`

- **ConfidenceResult.java** — 自信度分析结果
  - 字段：`guidance`(String)、`confidenceScore`(int)、`fillerCounts`(Map\<String, Integer\>)、`fillerDensity`(double)、`correctionCount`(int)

---

## Task 2: 新增 @Tool 工具类（3 个）

- **KnowledgeGapTool.java** (`agent/tool/`)
  - 依赖注入 `KnowledgeBaseService`
  - `@Tool(name = "detect_knowledge_gaps")` → 返回 `KnowledgeGapResult`
  - 参数：`question`, `user_answer`, `top_k`(optional)
  - 流程：调用 `kbService.searchAnswers(keyword, topK)` → 提取术语 → 词法对比 → 构建 guidance
  - 注意：`AnswerSearchResult.AnswerItem` 的答案字段是 `answer` 而非 `content`，需在概念提取时修正

- **TimeAllocationTool.java** (`agent/tool/`)
  - 无外部依赖
  - `@Tool(name = "analyze_time_allocation")` → 返回 `TimeAllocationResult`
  - 参数：`answers`（JSON 字符串，解析为 List\<QAPair\>）
  - 常量：CHARS_PER_MINUTE=250, IDEAL 60-120s
  - 仅返回评估枚举值（GOOD/ACCEPTABLE/TOO_SHORT/TOO_LONG），评语由 LLM 生成

- **ConfidenceTool.java** (`agent/tool/`)
  - 无外部依赖
  - `@Tool(name = "analyze_confidence")` → 返回 `ConfidenceResult`
  - 参数：`text`（完整回答文本）
  - 统计犹豫词（"嗯/那个/就是/然后"等）密度 + 自我修正模式 → 0-100 自信度评分
  - 评分公式暂用需求文档中的系数，标注 TODO

## Task 3: 扩展 Entity 与 DB 变更

- **InterviewQuestion.java** 新增字段：
  - `confidenceScore` (Integer) — 自信度评分 0-100
  - `timeAllocation` (String) — 时间分配评估枚举值

- **SQL 变更**：生成 ALTER TABLE 脚本到 `Documents/Sql变更/`
  ```sql
  ALTER TABLE op_interview_question
      ADD COLUMN confidence_score INT DEFAULT NULL COMMENT '自信度评分 0-100',
      ADD COLUMN time_allocation VARCHAR(32) DEFAULT NULL COMMENT '时间分配评估: GOOD/TOO_SHORT/TOO_LONG';
  ```

## Task 4: 更新 AgentFactory

- **构造函数**：接受 3 个新工具参数（KnowledgeGapTool, TimeAllocationTool, ConfidenceTool）
- **buildToolkit()**：在 `interview` 分组注册 3 个新工具
- **buildTechEvalAgent()**：
  - tools 从 `["search_answers", "analyze_answer", "search_questions"]` 扩展为 `["search_answers", "analyze_answer", "search_questions", "detect_knowledge_gaps"]`
  - inlineAgentsBody 增加第 4 步：调用 detect_knowledge_gaps 对比用户答案与标准答案，标记知识盲区
- **buildExprEvalAgent()**：
  - tools 从 `["analyze_answer"]` 扩展为 `["analyze_answer", "analyze_time_allocation", "analyze_confidence"]`
  - inlineAgentsBody 增加步骤 2-4：调用时间分配分析和自信度分析
- **buildPermissions()**：新增 3 条 ALLOW 规则（`detect_knowledge_gaps`, `analyze_time_allocation`, `analyze_confidence`）
- **buildSystemPrompt()**：在 Tool output interpretation 段落新增 3 个工具的指引

---

## Task 5: 扩展 InterviewAnalysisService

- **saveAnalysis()**：方法签名增加 `confidenceScore` 和 `timeAllocation` 参数，保存到 question 实体
- **buildDimensions()**：从 3 维扩展到 6 维（新增 knowledgeGapScore, timeAllocationScore, confidenceScore）
  - knowledgeGapScore：基于 `coverageScore` 映射（知识盲区 = 100 - coverage 近似值）
  - timeAllocationScore：按枚举值映射（GOOD=85, ACCEPTABLE=70, TOO_SHORT=50, TOO_LONG=50, null=60）
  - confidenceScore：直接使用 `InterviewQuestion.confidenceScore`
- **buildDetails()**：每道题的 detail map 增加 `confidenceScore` 和 `timeAllocation`
- **buildImprovements()**：基于新维度增加改进建议（低自信度 → 建议练习流畅表达，时间过短 → 建议扩展回答）
