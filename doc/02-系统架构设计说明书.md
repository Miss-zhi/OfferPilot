## 2. 工具定义（13 个本地 @Tool + MCP 联网搜索）

每个工具都是 Agent 能调用的一个"动作"。分为**查询类**（只读）和**操作类**（改变状态或有外部成本）。检索类工具内部调用 KnowledgeBaseService，自动执行"公共库 + 用户私有库"联合检索（详见《03-详细设计说明书》中"多租户知识库设计"章节）。

### 2.1 简历解析工具

```java
/**
 * 简历解析工具
 *
 * Agent 什么时候调它：用户上传简历 PDF 时。
 * 内部实现：PDFBox 提取文字 → LLM 结构化解析。
 * 返回 POJO，框架自动序列化为 JSON 给 Agent。
 */
@Component
public class ResumeParseTool {

    @Tool(
        name = "parse_resume",
        description = "解析简历 PDF，提取结构化信息（教育背景、项目经历、技能清单、实习/工作经历）。" +
                      "当用户上传简历并要求分析时使用。",
        readOnly = true
    )
    public ResumeParseResult parseResume(
        @ToolParam(name = "file_path", description = "简历文件的路径，如 'uploads/resume_zhangsan.pdf'。")
        String filePath
    ) {
        // 1. PDFBox 提取文字
        String rawText = pdfExtractor.extract(filePath);

        // 2. LLM 结构化解析
        ResumeParseResult result = llmParser.parseResume(rawText);
        return result;  // 框架自动序列化为 JSON
    }
}

/** 简历解析结果 DTO */
public class ResumeParseResult {
    private String name;
    private String education;            // 如 "XX大学 计算机 本科"
    private List<String> skills;         // 如 ["Java", "Spring Boot", "MySQL"]
    private List<ProjectInfo> projects;
    private List<InternshipInfo> internships;
    // getter/setter 省略
}

public class ProjectInfo {
    private String name;
    private String techStack;
    private String description;
}

public class InternshipInfo {
    private String company;
    private String role;
    private String duration;
}
```

**所有 @Tool 方法都遵循同样的模式：** 定义返回 DTO → 方法返回 POJO → 框架自动序列化。不再手动拼 JSON 字符串。其他工具的返回 DTO 定义在 `dto/tool/` 包下（详见《04-实现与编码规范》中"项目工程结构"章节）。

```java
@Component
public class ResumeEvaluateTool {

/** 简历评估结果 DTO — 工具返回原始数据 + 评估指导，由 LLM 据此生成评分和建议 */
@Data
public class ResumeEvaluateResult {
    private String resumeText;                // 简历原文
    private String guidance;                 // 评估指导文本，指示 LLM 如何评估
    private Integer overallScore;            // 综合评分（0-100），由 LLM 生成，此处为 null
    private String summary;                  // 总结，由 LLM 生成，此处为 null
    private List<String> strengths;          // 优点，由 LLM 生成，此处为空
    private List<String> weaknesses;         // 不足，由 LLM 生成，此处为空
    private List<String> suggestions;        // 改进建议，由 LLM 生成，此处为空
}
```

**设计原则（工具与 LLM 职责分离）：**
- **工具职责**：提取简历文本、构建评估指导（告知 LLM 评估维度和格式要求）
- **LLM 职责**：根据指导文本，结合简历内容，自由生成评分、优点/不足、改进建议
- **为什么**：工具返回结构化数据（查询、计算、持久化），LLM 负责自然语言内容生成。硬编码评分规则和评语模板会限制系统的智能性和通用性

### 2.3 面试题检索工具（接入向量检索）

```java
@Component
public class QuestionSearchTool {

    private final KnowledgeBaseService knowledgeBase;

    public QuestionSearchTool(KnowledgeBaseService knowledgeBase) {
        this.knowledgeBase = knowledgeBase;
    }

    @Tool(
        name = "search_questions",
        description = "搜索面试题库。按关键词语义检索，支持按分类和难度过滤。" +
                      "返回题目、参考答案、评分标准。",
        readOnly = true
    )
    public QuestionSearchResult searchQuestions(
        @ToolParam(name = "keyword", description = "搜索关键词，如 'HashMap'、'Spring 事务'。")
        String keyword,

        @ToolParam(name = "category", description = "技术分类，如 'Java基础'、'MySQL'。留空不限。")
        String category,

        @ToolParam(name = "max_difficulty", description = "最大难度 1-5，0 不限。")
        int maxDifficulty,

        @ToolParam(name = "limit", description = "返回数量，默认 5。")
        int limit
    ) {
        // 内部自动检索公共题库 + 当前用户的私有题库
        List<KnowledgeHit> hits = knowledgeBase.searchQuestions(
            keyword, category, maxDifficulty, limit > 0 ? limit : 5);

        // 转换为 POJO，框架自动序列化
        return QuestionSearchResult.from(hits);
    }
}

/** 面试题检索结果 DTO */
@Data
public class QuestionSearchResult {
    private int total;
    private List<QuestionItem> questions;

    @Data
    public static class QuestionItem {
        private String content;               // 题目内容
        private String tags;                  // 标签，如 "Java基础,HashMap"
        private double relevanceScore;        // 语义相关度 0-1
    }

    public static QuestionSearchResult from(List<KnowledgeHit> hits) {
        QuestionSearchResult result = new QuestionSearchResult();
        result.setTotal(hits.size());
        result.setQuestions(hits.stream().map(h -> {
            QuestionItem item = new QuestionItem();
            item.setContent(h.getContent());
            item.setTags(h.getTags());
            item.setRelevanceScore(h.getScore());
            return item;
        }).toList());
        return result;
    }
}
```

### 2.6 面试回答分析工具

```java
@Component
public class AnswerAnalyzeTool {

    private final InterviewQuestionRepository questionRepo;

    public AnswerAnalyzeTool(InterviewQuestionRepository questionRepo) {
        this.questionRepo = questionRepo;
    }

    @Tool(
        name = "analyze_answer",
        description = "保存面试回答并获取评估指导，由你据此分析回答质量并生成评分和评语。" +
                      "当需要分析候选人的面试回答时使用。",
        readOnly = true
    )
    public AnswerAnalysisResult analyze(
        @ToolParam(name = "question", description = "面试问题原文。") String question,
        @ToolParam(name = "answer", description = "用户的回答内容。") String answer
    ) {
        // 1. 保存原始 QA 到 DB
        // 2. 构建评估指导文本（告知 LLM 评分维度和输出格式）
        // 3. 返回原始数据 + 指导文本，评分由 LLM 在对话中生成
        return new AnswerAnalysisResult(question, answer, buildGuidance(question, answer),
                null, null, null, List.of(), List.of(), null);
    }

    /** 构建评估指导文本，不包含任何硬编码评分规则或评语模板 */
    private String buildGuidance(String question, String answer) {
        return String.format("""
                请分析以下面试回答的质量：
                【面试问题】%s
                【用户回答】%s
                请从以下三个维度评分（0-100）并给出评语：
                1. 专业深度 2. 表达能力 3. 内容覆盖度
                请输出格式：分数 — 评语 / 亮点 / 不足 / 改进建议
                """, question, answer);
    }
}

/** 面试回答分析结果 DTO — 工具返回原始数据 + 评估指导，由 LLM 据此生成评分和评语 */
@Data
public class AnswerAnalysisResult {
    private String question;                  // 原始面试问题
    private String answer;                    // 用户回答原文
    private String guidance;                 // 评估指导文本，指示 LLM 如何分析
    private Integer techScore;               // 专业深度评分（0-100），由 LLM 生成，此处为 null
    private Integer exprScore;               // 表达能力评分（0-100），由 LLM 生成，此处为 null
    private Integer coverageScore;           // 内容覆盖度评分（0-100），由 LLM 生成，此处为 null
    private List<String> highlights;         // 亮点，由 LLM 生成，此处为空
    private List<String> weaknesses;         // 不足，由 LLM 生成，此处为空
    private String suggestion;               // 改进建议，由 LLM 生成，此处为 null
}
```

**设计原则（工具与 LLM 职责分离）：**
- **工具职责**：保存原始 QA 数据到 DB、构建评估指导（告知 LLM 评分维度和输出格式要求）
- **LLM 职责**：根据指导文本，结合问题和回答内容，自由生成三维度评分及亮点/不足/建议
- **为什么**：不硬编码关键词匹配、评分规则、评语模板。让 LLM 根据实际回答内容动态评估，适应任意行业和岗位

```java
        description = "获取下一个面试题的出题指导（含岗位、阶段、难度、历史题目），由你据此生成并提问。" +
                      "当进行模拟面试需要下一道题目时使用。",
        readOnly = true
    )
    public NextQuestionResult generateNextQuestion(
        @ToolParam(name = "context", description = "当前面试上下文，包括已有问答历史和面试方向。")
        String context
    ) {
        // 1. 解析 sessionId、统计已有题目数
        // 2. 提取面试岗位、确定面试阶段和难度
        // 3. 查询 DB 获取历史得分和已出题目
        // 4. 构建出题指导（不包含任何题目模板），由 LLM 据此自由生成面试题
        String guidance = buildGuidance(role, category, difficulty, questionNumber, avgScore);
        return new NextQuestionResult(guidance, category, difficulty, role, questionNumber, avgScore, prevQuestions);
    }

    /** 构建出题指导文本 — 仅提供上下文信息，不包含硬编码题目 */
    private String buildGuidance(String role, String category, String difficulty,
                                  int questionNumber, Double avgScore) {
        return "请为" + role + "岗位生成一道" + category
                + "类面试题，难度" + difficulty
                + "，这是第" + questionNumber + "题。"
                + "请根据岗位特点和面试阶段自由发挥，直接输出题目文本。";
    }
}

/** 模拟面试出题结果 DTO — 工具返回出题指导，由 LLM 据此生成实际面试题 */
@Data
public class NextQuestionResult {
    private String guidance;                  // LLM 出题指导，如 "请为产品经理岗位生成一道专业技能类面试题..."
    private String category;                  // 面试阶段：自我介绍/专业技能/项目经验/情景分析/行为面试/职业规划
    private String difficulty;               // 难度：easy/medium/hard
    private String interviewRole;             // 面试岗位/方向
    private Integer questionNumber;           // 当前是第几题（1-based）
    private Double averageScore;             // 历史均分（0-100），无历史时为 null
    private List<String> previousQuestions;   // 前几题的文本，供 LLM 避免出重复题
}
```

**设计原则（工具与 LLM 职责分离）：**
- **工具职责**：查 DB（session/历史题目/得分）→ 计算进度/难度/阶段 → 输出出题指导
- **LLM 职责**：根据指导信息（岗位、阶段、难度、历史题目），结合上下文自由生成个性化面试题
- **为什么**：不硬编码任何题目模板——不同岗位（程序员/产品/金融/医疗）所需的面试题完全不同。LLM 根据 `interviewRole` 动态生成，确保系统适用于所有行业

### 2.12 联网搜索工具（MCP 协议接入）

联网搜索不手写 Java 代码，而是通过 **MCP 协议** 接入 OpenWebSearch MCP Server。Agent 启动时自动发现并注册搜索工具，业务代码零改动。

**为什么用 MCP 而不是 @Tool：**
- 国内用百度搜索引擎，无障碍，不依赖海外 API
- 零代码集成，配置即用，面试时讲"通过 MCP 协议动态接入搜索服务"更有技术含量
- 和 AgentScope v2 的架构理念一致——工具来源可插拔（本地 @Tool + 远程 MCP Server 混合）

**MCP Server 选型：OpenWebSearch**

内置支持百度、Bing、DuckDuckGo，还专门适配了 CSDN、掘金等国内开发者社区的内容抓取。

**配置方式（`workspace/tools.json`）：**

```json
{
  "mcpServers": {
    "web-search": {
      "transport": "stdio",
      "command": "npx",
      "args": ["-y", "open-web-search-mcp"],
      "env": {
        "SEARCH_ENGINE": "baidu",
        "MAX_RESULTS": "5"
      }
    }
  }
}
```

Agent 启动时自动扫描 `workspace/tools.json`，连接 MCP Server，将 `web_search` 注册为自身工具。无需在 Java 代码中手动注册。

**Fallback 策略（确定性规则）：** 知识库检索工具的返回结果中，每个结果携带 `relevanceScore` 字段。Fallback 触发条件在 SysPrompt 中明确定义——**当 `total == 0`（无结果）或最高 `relevanceScore < 0.6`（相关性不足）时，Agent 必须调用 MCP `web_search` 从互联网补充信息**。这是确定性规则而非 LLM 自主决策，确保 fallback 行为可预测。

**Permission 控制：** MCP 工具与本地 @Tool 在权限系统中地位完全平等，可以直接在 Rule 中引用 `web_search` 工具名：

```java
PermissionContextState perms = PermissionContextState.builder()
    .addAllowRule("web_search", new PermissionRule("web_search", null,
        PermissionBehavior.ALLOW, "联网搜索直接放行"))  // 查询类工具，直接放行
    .build();
```

### 2.13 工具注册与分发策略

**架构原则**：AgentScope v2 的 `SubagentsMiddleware` 要求子 Agent 使用的工具必须注册在主 Agent 的 Toolkit 上——子 Agent 在 spawn 时获得主 Toolkit 的**过滤副本**（按 `SubagentDeclaration.tools()` 白名单筛选）。因此，虽然主 Agent 的 SysPrompt 禁止直接调用业务工具，但所有 13 个 `@Tool` 仍需注册在主 Toolkit 上。

所有 Tool 类标注 `@Component`，由 Spring 容器管理生命周期和依赖注入。`AgentFactory` 通过构造器注入所有 Tool Bean，注册到 Toolkit：

```java

// 在 buildAgent() 中注册到 Toolkit：
//
// 【架构说明】13 个业务工具全部注册在主 Agent 的 Toolkit 上，原因：
//   AgentScope v2 的 SubagentsMiddleware 在 spawn 子 Agent 时，
//   从主 Agent 的 Toolkit 中按 SubagentDeclaration.tools() 白名单
//   筛选工具，为每个子 Agent 构建独立的过滤 Toolkit。
//   因此主 Agent 必须持有全部工具，但主 Agent 的 SysPrompt 严禁
//   它直接调用业务工具——主 Agent 只使用 spawn/resume_subagent 和
//   元工具（reset_equipped_tools、todo_write）。
//
Toolkit toolkit = new Toolkit();
// 简历相关
toolkit.registerTool(resumeParseTool);          // @Component，注入 LlmParser
toolkit.registerTool(resumeEvaluateTool);       // @Component，注入 LlmEvaluator
// 题库相关（内部注入 KnowledgeBaseService，自动执行多租户检索）
toolkit.registerTool(questionSearchTool);
toolkit.registerTool(answerSearchTool);
// 公司调研
toolkit.registerTool(companySearchTool);
// 面试分析
toolkit.registerTool(answerAnalyzeTool);        // @Component，注入 LlmEvaluator
toolkit.registerTool(audioTranscribeTool);      // @Component，注入 AsrService
// 模拟面试
toolkit.registerTool(mockInterviewTool);        // @Component，注入 KnowledgeBaseService
// 学习规划
toolkit.registerTool(resourceSearchTool);       // @Component，注入 KnowledgeBaseService
toolkit.registerTool(progressTrackTool);        // @Component，注入 ProgressService
// 薪资谈判
toolkit.registerTool(salaryTool);               // @Component，注入 KnowledgeBaseService
// 联网搜索：通过 MCP 协议自动注册（workspace/tools.json 配置），无需手动注册

// ⚠️ 工具分组已废弃：分组降低 token 消耗的初衷是正确的，但主 Agent 本身不
// 调用业务工具，子 Agent 各自持有独立的过滤 Toolkit（仅含 2-3 个工具），
// 因此分组在主 Agent 上没有意义。工具白名单由 SubagentDeclaration.tools() 管理。
// 保留 registerMetaTool()：主 Agent 通过 reset_equipped_tools 可在对话中
// 动态启禁工具组（尽管主 Agent 不应调用业务工具，但元工具是通用的调度能力）。
toolkit.registerMetaTool();

// Agent 构建时指定 workspace，自动扫描 tools.json 并接入 MCP 搜索工具
HarnessAgent agent = HarnessAgent.builder()
    .name("offerPilotCoach")
    .sysPrompt("你是 OfferPilot 面试诊断助手...")

```

### 2.14 Tool 线程安全规范（CR 检查清单）

由于 13 个 `@Tool` 类都是 Spring 单例 Bean，而多个用户的 Agent 实例可能并发调用同一工具实例，每个 `@Tool` 方法必须满足以下线程安全约束：

| 检查项 | 要求 | 违反示例 |
|:---|:---|:---|
| 禁止实例可变状态 | `@Tool` 方法不得修改 `this.xxx` 成员变量 | `this.lastQuery = keyword;` ❌ |
| 多租户隔离 | 所有数据访问必须通过 `KnowledgeBaseService`，由其内部根据 `userId` 过滤 | 直接查全局表不带 `userId` ❌ |
| 依赖只读注入 | 构造器注入的依赖（`KnowledgeBaseService`、`LlmEvaluator` 等）必须是线程安全的 | 注入带写缓存的 Service 需验证 ❌ |
| 方法参数驱动 | 工具行为仅由 `@ToolParam` 参数决定，不依赖外部隐式状态 | 从 ThreadLocal 读取 userId ✅（框架注入） |

**正确示例（线程安全的 Tool）：**

```java
@Component
public class QuestionSearchTool {
    private final KnowledgeBaseService knowledgeBase;  // 构造器注入，只读依赖

    @Tool(name = "search_questions", readOnly = true)
    public QuestionSearchResult searchQuestions(
        @ToolParam(name = "keyword") String keyword,   // 行为仅由参数驱动
        @ToolParam(name = "category") String category) {
        // ✅ 内部调用 knowledgeBase.search(userId=RuntimeContext.getUserId(), ...)
        return knowledgeBase.searchQuestions(keyword, category);
    }
}
```

**运行时多租户隔离机制：** 每个 `@Tool` 方法执行时，AgentScope 框架自动将 `RuntimeContext`（含 `userId`、`sessionId`）注入 Reactor 上下文。`KnowledgeBaseService` 从上下文中提取 `userId` 进行数据隔离。Tool 开发者无需手动传递 `userId`。

## 4. 多 Agent 协作设计（1 主 + 7 子）

> **架构说明**：AgentScope v2 的 `SubagentsMiddleware` 在 spawn 子 Agent 时，从主 Agent 的 Toolkit 中按 `SubagentDeclaration.tools()` 白名单筛选工具，为每个子 Agent 构建**独立的过滤 Toolkit 副本**。这意味着：

## 5. 用户认证与权限系统

### 5.1 整体设计

系统分两种角色：**USER**（普通用户）和 **ADMIN**（管理员）。认证采用 Spring Security + JWT 无状态方案。

```
注册 → 登录 → 获取 JWT Token → 每次请求携带 Token
                                    ↓
                           JwtAuthFilter 解析 Token
                                    ↓
                        SecurityContext 设置用户身份
                                    ↓
                    Controller 通过 @PreAuthorize 鉴权
```

### 5.4 JWT 工具类

```java
@Component
public class JwtTokenProvider {

    @Value("${app.jwt.secret}")
    private String secret;

    @Value("${app.jwt.expiration-ms:86400000}")  // 默认 24 小时
    private long expirationMs;

    /** 从用户名生成 Token */
    public String generateToken(String username) {
        return Jwts.builder()
            .setSubject(username)
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + expirationMs))
            .signWith(Keys.hmacShaKeyFor(secret.getBytes()), SignatureAlgorithm.HS256)
            .compact();
    }

    /** 从 Token 解析用户名 */
    public String extractUsername(String token) {
        return parseClaims(token).getSubject();
    }

    /** 验证 Token 是否有效 */
    public boolean validateToken(String token, String username) {
        try {
            String extracted = extractUsername(token);
            return extracted.equals(username) && !isTokenExpired(token);
        } catch (JwtException e) {
            return false;
        }
    }

    private Claims parseClaims(String token) {
        return Jwts.parserBuilder()
            .setSigningKey(Keys.hmacShaKeyFor(secret.getBytes()))
            .build()
            .parseClaimsJws(token)
            .getBody();
    }

    private boolean isTokenExpired(String token) {
        return parseClaims(token).getExpiration().before(new Date());
    }
}
```

**Token 黑名单机制：** 为支持用户主动登出和强制 Token 失效，JWT 生成时携带 `jti`（JWT ID）唯一标识。登出时将该 jti 写入 `op_token_blacklist` 表。`JwtAuthenticationFilter` 验证 Token 时额外检查黑名单：

```java
// JwtTokenProvider.generateToken() 增强版
public String generateToken(String username) {
    String jti = UUID.randomUUID().toString();
    return Jwts.builder()
        .setSubject(username)
        .setId(jti)                                    // jti 用于黑名单校验
        .setIssuedAt(new Date())
        .setExpiration(new Date(System.currentTimeMillis() + expirationMs))
        .signWith(Keys.hmacShaKeyFor(secret.getBytes()), SignatureAlgorithm.HS256)
        .compact();
}

// JwtAuthenticationFilter 增强：检查黑名单
if (tokenBlacklistService.isBlacklisted(jti)) {
    response.setStatus(401);
    return;  // Token 已失效
}
```

**定时清理：** 每天凌晨 3 点清理 `op_token_blacklist` 中已过期的记录，防止黑名单表无限增长。

### 5.5 JWT 认证过滤器

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtTokenProvider jwtTokenProvider;
    private final UserDetailsService userDetailsService;

    public JwtAuthenticationFilter(JwtTokenProvider jwtTokenProvider,
                                   UserDetailsService userDetailsService) {
        this.jwtTokenProvider = jwtTokenProvider;
        this.userDetailsService = userDetailsService;
    }

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain filterChain) throws ServletException, IOException {

        String authHeader = request.getHeader("Authorization");

        // 没有 Token 直接放行，让 Security 链决定是否需要认证
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        String token = authHeader.substring(7);
        String username = jwtTokenProvider.extractUsername(token);

        if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);

            if (jwtTokenProvider.validateToken(token, userDetails.getUsername())) {
                UsernamePasswordAuthenticationToken authToken =
                    new UsernamePasswordAuthenticationToken(
                        userDetails, null, userDetails.getAuthorities());
                authToken.setDetails(
                    new WebAuthenticationDetailsSource().buildDetails(request));
                SecurityContextHolder.getContext().setAuthentication(authToken);
            }
        }

        filterChain.doFilter(request, response);
    }
}
```

### 5.6 认证 Service 和 Controller

```java
@Service
public class AuthService {

    private final AppUserRepository userRepository;
    private final PasswordEncoder passwordEncoder;
    private final JwtTokenProvider jwtTokenProvider;

    public AuthService(AppUserRepository userRepository,
                       PasswordEncoder passwordEncoder,
                       JwtTokenProvider jwtTokenProvider) {
        this.userRepository = userRepository;
        this.passwordEncoder = passwordEncoder;
        this.jwtTokenProvider = jwtTokenProvider;
    }

    /** 注册 */
    public AuthResponse register(@Valid RegisterRequest req) {
        if (userRepository.findByUsername(req.getUsername()).isPresent()) {
            throw new DuplicateUserException();                 // 409
        }

        AppUser user = new AppUser();
        user.setUserId(generateUserId());
        user.setUsername(req.getUsername());
        user.setPasswordHash(passwordEncoder.encode(req.getPassword()));
        user.setEmail(req.getEmail());
        user.setRole(UserRole.USER);                            // 枚举，非硬编码字符串
        user.setEnabled(true);
        userRepository.save(user);

        String token = jwtTokenProvider.generateToken(user.getUsername());
        return new AuthResponse(token, user.getUserId(), user.getUsername(), UserRole.USER);
    }

    /** 登录 */
    public AuthResponse login(@Valid LoginRequest req) {
        AppUser user = userRepository.findByUsername(req.getUsername())
            .orElseThrow(InvalidCredentialsException::new);     // 401

        if (!passwordEncoder.matches(req.getPassword(), user.getPasswordHash())) {
            throw new InvalidCredentialsException();            // 401
        }

        if (!user.getEnabled()) {
            throw new AccountDisabledException();               // 403
        }

        user.setLastLoginAt(Instant.now());
        userRepository.save(user);

        String token = jwtTokenProvider.generateToken(user.getUsername());
        return new AuthResponse(token, user.getUserId(), user.getUsername(), user.getRole());
    }
    /** 登出（将 Token 加入黑名单） */
    public void logout(String jti, String userId) {
        tokenBlacklistService.blacklist(jti, userId, "LOGOUT");
    }
}

@RestController
@RequestMapping("/api/v1/auth")
public class AuthController {

    private final AuthService authService;

    public AuthController(AuthService authService) {
        this.authService = authService;
    }

    @PostMapping("/register")
    public ResponseEntity<ApiResponse<AuthResponse>> register(
            @RequestBody @Valid RegisterRequest req) {
        return ResponseEntity.status(201)
            .body(ApiResponse.success(authService.register(req)));
    }

    @PostMapping("/login")
    public ResponseEntity<ApiResponse<AuthResponse>> login(
            @RequestBody @Valid LoginRequest req) {
        return ResponseEntity.ok(ApiResponse.success(authService.login(req)));
    }

    @PostMapping("/logout")
    public ResponseEntity<ApiResponse<Void>> logout(HttpServletRequest request) {
        String jti = (String) request.getAttribute("jti");  // Filter 中设置
        String userId = (String) request.getAttribute("userId");
        authService.logout(jti, userId);
        return ResponseEntity.ok(ApiResponse.success(null));
    }
}
```

### 5.8 定时任务设计

为满足"定期更新面经数据"的业务需求，系统使用 Spring `@Scheduled` 实现轻量级定时任务调度，同时将执行日志记录到 `op_scheduled_task_log` 表，便于排查和监控。

**已规划的定时任务：**

| 任务名称 | Cron | 说明 |
|:---|:---|:---|
| `refreshCompanyInterviews` | `0 0 3 * * SUN` | 每周日凌晨 3 点，通过 MCP `web_search` 更新公司面经数据 |
| `cleanExpiredTokens` | `0 0 3 * * *` | 每天凌晨 3 点，清理 `op_token_blacklist` 中已过期的记录 |
| `cleanStaleSessions` | `0 0 4 * * *` | 每天凌晨 4 点，清理 Redis 中过期的会话缓存 |

```java
@Configuration
@EnableScheduling
public class ScheduledTaskConfig {

    private final ScheduledTaskLogRepository taskLogRepo;
    private final KnowledgeBaseService kbService;
    private final TokenBlacklistService blacklistService;

    @Scheduled(cron = "0 0 3 * * SUN")
    public void refreshCompanyInterviews() {
        ScheduledTaskLog log = startTask("refreshCompanyInterviews");
        try {
            int updated = kbService.refreshCompanyInterviews();
            completeTask(log, "更新了 " + updated + " 条公司面经数据");
        } catch (Exception e) {
            failTask(log, e);
        }
    }

    @Scheduled(cron = "0 0 3 * * *")
    public void cleanExpiredTokens() {
        ScheduledTaskLog log = startTask("cleanExpiredTokens");
        try {
            int cleaned = blacklistService.cleanExpired();
            completeTask(log, "清理了 " + cleaned + " 条过期 Token");
        } catch (Exception e) {
            failTask(log, e);
        }
    }

    private ScheduledTaskLog startTask(String name) { /* 记录开始 */ }
    private void completeTask(ScheduledTaskLog log, String summary) { /* 记录成功 */ }
    private void failTask(ScheduledTaskLog log, Exception e) { /* 记录失败 */ }
}
```

**Note:** 定时任务在当前版本中为"远期规划"功能——DDL 表已定义，`@Scheduled` 注解和 `ScheduledTaskConfig` 骨架已就位，具体业务逻辑（如 MCP `web_search` 批量抓取）在面经数据积累到一定量后再激活。面试时可以说"架构上预留了定时任务调度能力，DDL 中定义了执行日志表，符合企业级运维规范"。
