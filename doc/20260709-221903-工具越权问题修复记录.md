# 工具越权问题修复 — 职责分离重构

## 背景

OfferPilot 作为智能体系统，存在工具类越权承担 LLM 职责的问题：多个 `@Tool` 工具类硬编码了自然语言内容（题目、评语、建议），导致系统无法适配非程序员岗位。

## 核心原则

> **工具提供结构化数据，LLM 负责自然语言生成**

## 变更清单

### P0 — AnswerAnalyzeTool（最严重）

**文件**: `src/main/java/com/tutorial/offerpilot/agent/tool/AnswerAnalyzeTool.java`

| 删除 | 原因 |
|------|------|
| `TECH_KEYWORD` Pattern（"算法\|数据结构\|设计模式\|..."） | 只认识程序员关键词，产品经理回答得 0 分 |
| `evaluateTechScore()` — 按关键词命中数打分 | 粗糙规则，不如 LLM 理解能力强 |
| `evaluateExpression()` — 检查"首先/第一/总结" | 硬编码规则 |
| `evaluateCoverage()` — 检查"例如/优点/缺点" | 硬编码规则 |
| `extractHighlights()` — "技术关键词覆盖较好"等 4 条 | 硬编码评语 |
| `extractWeaknesses()` — "建议使用STAR法则"等 4 条 | 硬编码评语 |
| `generateSuggestion()` — 3 条硬编码建议 | 千篇一律 |

**改为**: 工具保存原始 QA 到 DB → 生成评估指导文本 → LLM 据此自由生成评分/评语/建议。

### P0 — ResumeService（评估部分）

**文件**: `src/main/java/com/tutorial/offerpilot/service/ResumeService.java`

| 删除 | 原因 |
|------|------|
| `calculateOverallScore()` — 按关键词"项目/负责/技能/技术"打分 | 纯技术导向评分 |
| `generateSummary()` — "简历质量优秀/良好/需改进" | 硬编码总结 |
| `extractStrengths()` — "简历内容充实"等 | 硬编码优点 |
| `extractWeaknesses()` — "简历内容过短"等 | 硬编码不足 |
| `generateSuggestions()` — "使用STAR法则"等 | 硬编码建议 |
| `extractSkills()` 硬编码技能列表 | 改为模式匹配（"熟练掌握XXX"、"技能：XXX"） |

**保留**: `parseResume()`（正则提取姓名/邮箱/电话/教育/项目等结构化字段）— 这是工具该做的文本解析。

### P1 — MockInterviewTool

**文件**: `src/main/java/com/tutorial/offerpilot/agent/tool/MockInterviewTool.java`

| 删除 | 原因 |
|------|------|
| `QUESTION_TEMPLATES` — 30 道硬编码模板 | 无论怎么改模板都是写死的 |
| `selectQuestion()` — 模板选填逻辑 | 出题应由 LLM 自由发挥 |
| `buildReason()` — 拼接原因文本 | LLM 不需要这个 |

**改为**: 工具查 DB（session/历史题目/得分）→ 计算进度/难度/阶段 → 返回出题指导（含岗位、类别、难度、题号、历史题目、均分）→ LLM 自由生成面试题。

### DTO 变更

**NextQuestionResult**:
- 删除: `question`, `reason`
- 新增: `guidance`, `interviewRole`, `questionNumber`, `averageScore`, `previousQuestions`

**AnswerAnalysisResult**:
- 新增: `question`, `answer`, `guidance`
- 保留: `techScore`, `exprScore`, `coverageScore`, `highlights`, `weaknesses`, `suggestion`（均改为 null/空，由 LLM 填充）

**ResumeEvaluateResult**:
- 新增: `resumeText`, `guidance`
- 保留: `overallScore`, `summary`, `strengths`, `weaknesses`, `suggestions`（均改为 null/空）

### 系统提示词

**文件**: `src/main/java/com/tutorial/offerpilot/agent/AgentFactory.java`

新增 `IMPORTANT — Tool output interpretation` 段落，明确告知 LLM：
- `generate_next_question` 返回指导而非现成题目，LLM 必须自行生成
- `analyze_answer` 返回原始 QA + 指导，LLM 必须自行评估
- `evaluate_resume` 返回简历文本 + 指导，LLM 必须自行评估
